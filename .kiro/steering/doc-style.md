# 文档风格

所有技术文档、知识梳理、笔记，遵循以下原则：

## 核心

1. **第一性原理**：从"解决什么问题"出发，不从定义/表象入手
2. **直击本质**：一句话说清本质，类比辅助，不绕弯
3. **场景驱动**：绑定具体场景（"在 IoT App 中..."），不做脱离场景的科普
4. **问题驱动**：每个章节以问题开头（为什么？怎么做？区别？）

## 结构

5. **先骨架，后支干，再血肉**
6. **目录放最前面，要细到每个小节**
7. **概念速查表**放文件末尾
8. **对比用表格**，不用大段文字

## 粒度

9. **不是说明书**：不罗列 API，而是"做什么动作 → 用什么 → 为什么 → 效果"
10. **抓本质不堆细节**：AI 时代不需要记具体代码，要知道关键 API 名称、核心参数、作用
11. **万变不离其宗**：复杂话题先给统一本质模型，再展开具体手段
