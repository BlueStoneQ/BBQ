# 文档风格

所有技术文档、知识梳理、笔记，遵循以下原则：

## 核心

1. **第一性原理**：从"解决什么问题"出发，不从定义/表象入手
2. **直击本质**：一句话说清本质，类比辅助，不绕弯
3. **场景驱动**：绑定具体场景（"在 IoT App 中..."），不做脱离场景的科普
4. **问题驱动**：每个章节以问题开头（为什么？怎么做？区别？）
5. **结论先行**：最佳实践/结论/总结放在文档最前面，详细推导和注释放后面

## 结构

5. **先骨架，后支干，再血肉**
6. **目录放最前面，要细到每个小节**
7. **概念速查表**放文件末尾
8. **对比用表格**，不用大段文字

## 粒度

9. **不是说明书**：不罗列 API，而是"做什么动作 → 用什么 → 为什么 → 效果"
10. **抓本质不堆细节**：AI 时代不需要记具体代码，要知道关键 API 名称、核心参数、作用
11. **万变不离其宗**：复杂话题先给统一本质模型，再展开具体手段


## 代码示例

12. **示例前后连贯**：同一篇文档中的代码示例使用同一个贯穿案例（同一套变量名/类型名/数据模型），不要每个章节换一个例子
13. **命名不产生歧义**：示例中的命名避免与技术术语撞车（如不用 `post` 做数据模型名，容易和 HTTP POST 混淆）
14. **顺序对应**：多个代码块之间如果有映射关系（如定义→实现→调用），保持相同的字段/方法顺序，让读者从上到下扫一遍就能对齐
