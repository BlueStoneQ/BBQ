# Git Conventions

## Commit Message

- Always write commit messages in **English**
- Use conventional commits format: `type: short description`
- Types: feat, fix, docs, refactor, chore, style, test

## Data Desensitization

- **Never** use real company names in documents or commit messages
- Use code names or abbreviations (e.g. JY, XM, XC, XT, MT, DFGX)
- Same rule applies to personal information (real names, phone numbers, emails, etc.)
- Avoid sensitive Chinese terms like "面试", "简历" etc. in documents — use English equivalents (e.g. prep, checklist, review)
