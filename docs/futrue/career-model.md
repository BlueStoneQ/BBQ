## 一、当前能力模型

```
1. 前端 / 大前端 / 全栈 ← 10 年积累，地基
2. 大前端工程化 ← 分 Bundle / 热更新 / CLI / CI/CD / 质量体系
3. AI Agent ← Mako 框架 / MCP 全链路 / AI 辅助研发
4. 3D / 图形编辑器 ← 接下来持续积累的方向
```

**核心定位**：不是"写页面的前端"，是"跨端框架开发者 + 工程化架构师 + AI 工具构建者"。

## 二、AI 冲击分析

### AI Coding 替代的是什么

写 CRUD、写页面、写组件、写接口——标准化的、有明确输入输出的编码工作。

### AI Coding 替代不了的是什么

- 架构决策（"该不该分 Bundle？怎么分？"）
- 系统设计（"BLE 通信层怎么和 RN 桥接？"）
- 性能归因（"卡了，瓶颈在哪一层？"）
- 工程化体系建设（"怎么让 10 个人的团队高效协作？"）
- 跨层问题定位（JS → C++ → Android → 硬件）
- 物理世界交互（硬件调试、设备通信稳定性）

### 抽象层越高，替代越快

```
被冲击大：写页面、调样式、用框架写业务
被冲击中：Web 3D（工具层被简化，但图形学深度和创意仍需人）
被冲击小：造框架、设计 Bridge、做包体优化、建工程化体系
被冲击最小：嵌入式/硬件调试（物理世界 AI 做不了）
```

**当前位置**：在"被冲击小"的那一侧。不是写业务代码的人，是设计系统、解决复杂问题、建工程化体系的人。

## 三、四条线的定位与判断

| 线 | 定位 | AI 冲击 | 时间维度 | 判断 |
|---|------|---------|---------|------|
| 1. 前端/全栈 | 地基 | 纯编码被替代，架构不会 | 持续 | 保持，不用额外投入 |
| 2. 工程化 | 稳定价值 | 不会被替代（体系建设是人的活） | 持续 | 保持，是差异化 |
| 3. AI Agent | 当前差异化 | 方向对但格局变化快 | 1-3 年窗口 | 当前加分项，不确定能否作为 5 年赛道 |
| 4. 3D/图形 | 长坡厚雪 | 中等冲击（工具简化，深度不变） | 3-5 年积累 | **长期投入方向** |

### 为什么 3 不确定能作为 5 年赛道

- AI Agent 框架格局可能 2-3 年内被大厂收编（OpenAI/Anthropic/Google 都在做）
- 开源 Agent 框架的窗口期有限
- 但"用 AI Agent 提效"的能力是永久的——造框架可能过时，用框架不会

### 为什么 4 是长坡厚雪

- 门槛高：图形学 + 数学 + 架构 + 性能优化，不容易被大量人涌入
- AI 生成 3D 在爆发：但"展示/编辑/交互"仍需 Web 3D 工程师
- 应用场景扩大：电商 3D、数字孪生、AIGC 3D、AR/VR
- 经验壁垒高：越积累越深，不是学几天就能做的

## 四、3D/图形方向深度分析

### 条件：不能只停在 demo 层面

要深入到：
- 渲染管线优化（draw call/LOD/遮挡剔除）
- 编辑器架构（状态机/插件化/Command 模式）
- 大规模场景管理
- 和 AI 结合（AI 生成 3D + Web 展示/编辑）

### 和当前能力的协同

- 框架设计能力 + 3D → 3D 编辑器架构（稀缺组合）
- 性能优化 + 3D → 渲染性能优化（核心痛点）
- AI Agent + 3D → AI 驱动的 3D 生成工具链（前沿方向）
- 工程化 + 3D → 3D 项目的工程化体系（大部分 3D 项目工程化很差）

### 竞争格局

- 会 3D 的前端不多（门槛劝退）
- 会 3D + 会架构的更少
- 会 3D + 会架构 + 会 AI 的极少
- 不可替代性强 → 稳定性好

## 五、端侧 AI 延伸路径

### 不需要"转行"，自然延伸

```
当前：大前端架构 + IoT + AI Agent + RN↔Native
延伸：+ 端侧推理（TFLite/ONNX）+ 模型集成 + 多模态交互
```

### 定位：端侧 AI 应用集成层（不是模型优化层）

- 把模型部署到设备上
- 优化推理性能（线程管理/内存控制/按需加载）
- 设计 AI 功能的产品体验
- 封装成 TurboModule 给上层用

### 和"调 SDK"的区别

价值不在"集成某个 OCR SDK"，而在"设计端侧 AI 基础设施"：
- 模型怎么按需加载
- 多模型怎么调度
- 模型怎么热更新
- 推理结果怎么和业务串联

### 优先级

中长期方向（1-2 年），不是当前紧急事项。入职 IoT 公司后自然接触。

## 六、结论与行动优先级

### 短期（现在 ~ 入职）

- 专注 1+2+3：拿到好 offer（RN 架构 / 大前端工程化 / AI Agent 加分）
- 开源项目：XRN + Mako 作为能力证明

### 中期（入职后 1 年）

- 在 IoT 场景自然接触端侧 AI
- 开始 3D/图形编辑器项目（mindcraft），持续积累
- 工作中深化 RN 架构 + 工程化体系

### 长期（3-5 年）

- 3D/图形 成为核心差异化（长坡厚雪）
- 端侧 AI 作为补充能力
- 定位：**"懂 3D + 懂架构 + 懂 AI + 懂工程化"的复合型技术专家**

### 一句话

不需要焦虑"被替代"。保持在"AI 替代不了"的那一侧（架构/工程化/跨层/3D 深度），同时用 AI 放大自己的产出。
