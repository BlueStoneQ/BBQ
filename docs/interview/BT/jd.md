# BT - 大前端（红果/番茄）

## Responsibilities

1. 负责小说相关 B/C 端产品 **Web/Hybrid/Wap/小程序/Flutter** 的前端开发
2. 高质量设计和编码，承担**重点难点技术攻坚**，产出清晰技术方案并落地
3. 参与业务场景（**编辑器、阅读器、音/视频播放器**）的设计、开发与优化
4. **性能优化**，持续挖掘性能瓶颈，打造极致产品品质和用户体验
5. 前端**工程化、标准化、通用组件、效率平台**开发，打造高效研发工作流

## Requirements

1. 本科及以上，计算机相关，扎实基础（**数据结构、算法、设计模式**）
2. 熟练主流框架：**React、TypeScript、Node.js、小程序、Flutter** 优先
3. 熟悉前端工程化：研发提效、性能优化、稳定性、工程架构
4. 良好方案设计能力 + 编码习惯，能自主完成技术方案设计及落地
5. 产品意识 + 技术敏感度，关注业界最新前端技术

## Bonus

1. 服务端（Go/Node.js）开发经验
2. **跨端（Android/iOS/Flutter/小程序）**开发经验
3. 热衷工具软件产品，提高个人/团队生产力
4. **前端社区活跃、有开源项目**
5. 小说重度用户

## Notes

- **字节必考算法**，至少准备 20 道中等题
- 开源项目是核心亮点（Mako + XRN），主动提
- 性能优化 + 工程化是你的强项，和 JD 高度匹配
- Flutter 没深度实践，被问到讲选型对比（为什么选 RN 不选 Flutter）
- 编辑器/阅读器场景可以用快应用 IDE + ECharts 大数据渲染类比
- 字节面试风格：算法 + 项目深挖 + 系统设计，每轮 45~60min
