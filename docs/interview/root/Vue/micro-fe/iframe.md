# iframe 做微前端：优缺点与选型

---

## 本质

iframe 是浏览器原生提供的**进程级隔离**方案——每个 iframe 有独立的 window、document、JS 执行环境、CSS 作用域。

**它是"隔离最强"的微前端方案，但也是"体验最差"的。**

---

## 优点

| 优点 | 说明 |
|------|------|
| **隔离最彻底** | 独立 window/document，JS/CSS 天然互不干扰 |
| **零改造成本** | 子应用不需要导出生命周期、不需要改构建配置 |
| **技术栈完全无关** | 任何能独立跑的页面都能嵌 |
| **安全性好** | 同源策略天然防护，sandbox 属性可进一步限制 |
| **稳定** | 子应用崩溃不影响主应用（独立进程） |

---

## 缺点（为什么大家不爱用）

| 问题 | 具体表现 |
|------|---------|
| **URL 不同步** | iframe 内部路由变化不反映到浏览器地址栏，刷新丢失状态 |
| **UI 割裂** | 弹窗/遮罩被困在 iframe 内，无法覆盖全屏；滚动条独立 |
| **通信复杂** | 只能用 `postMessage`，数据必须可序列化，异步且笨重 |
| **性能开销** | 每个 iframe 独立进程/独立渲染管线，内存占用高 |
| **SEO 不友好** | 搜索引擎不索引 iframe 内容 |
| **白屏问题** | 每次切换 iframe 都要重新加载（无法保活，或保活占大量内存） |
| **高度自适应困难** | iframe 不能根据内容自动撑高，需要 JS 手动同步 |
| **共享状态困难** | 登录态/Token 需要手动传递或依赖同域 Cookie |

---

## 典型问题场景

```
场景 1：弹窗被裁剪
  子应用弹出 Modal → 只能在 iframe 框内显示
  → 遮罩盖不住主应用内容
  → 用户体验像"在小窗里操作"

场景 2：路由丢失
  用户在子应用里导航到 /activity/123
  → 浏览器地址栏还是主应用的 URL
  → 刷新页面 → 回到子应用首页 → 状态丢失

场景 3：通信成本
  主应用要传用户信息给子应用：
  // 主应用
  iframe.contentWindow.postMessage({ user, token }, '*');
  // 子应用
  window.addEventListener('message', (e) => { ... });
  // 每次都是异步 + 序列化/反序列化 + 来源校验
```

---

## 什么场景适合用 iframe

| 场景 | 为什么适合 |
|------|-----------|
| **嵌入第三方页面**（不可控的外部系统） | 无法改造，只能嵌 |
| **遗留系统集成**（10 年前的 jQuery 页面） | 改造成本 > 重写成本 |
| **安全沙箱**（用户上传的自定义页面/插件） | 需要强隔离防止恶意代码 |
| **独立工具页**（文档预览、报表、地图） | 交互少，不需要深度集成 |
| **快速过渡方案**（先跑起来，后续再迁移） | 零改造先上线 |

---

## 什么场景不该用 iframe

| 场景 | 为什么不行 |
|------|-----------|
| 多个子应用频繁切换 | 每次切换重新加载，白屏严重 |
| 子应用有全屏弹窗/遮罩需求 | UI 被困在 iframe 边界内 |
| 需要深度通信（共享状态） | postMessage 太重 |
| SPA 体验要求高 | 路由不同步、无法前进后退 |
| 移动端 | 滚动嵌套问题、性能更敏感 |

## iframe vs 其他方案速查

| 维度 | iframe | qiankun | Module Federation | Web Components |
|------|--------|---------|-------------------|----------------|
| 隔离强度 | ★★★★★ | ★★★☆☆ | ★☆☆☆☆ | ★★★☆☆ |
| 体验 | ★★☆☆☆ | ★★★★☆ | ★★★★★ | ★★★★☆ |
| 通信成本 | 高（postMessage） | 低（函数调用） | 低（直接 import） | 中（CustomEvent） |
| 改造成本 | 零 | 中 | 中 | 低 |
| 性能 | 差（多进程） | 中 | 好 | 中 |

---

## 一句话

> **iframe 是"真隔离"换"真割裂"——隔离最强但体验最差。适合嵌入不可控的外部系统或需要强安全沙箱的场景；不适合追求 SPA 体验的内部系统集成。**
