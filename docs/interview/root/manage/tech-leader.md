# 技术 Leader 四维管理体系

> 解决什么问题：面试中怎么讲"管理经验"？不是讲排期分活，是讲"我用什么手段解决了什么团队问题"。
>
> 本质：技术 TL = 通过方向把控 + 机制设计 + 工具提效 + 人才培养，让团队持续高产出。
>
> 核心原则：讲"建体系"不讲"做管理"，讲"量化结果"不讲"日常动作"。

"我有两种管理经验：一是正式带团队（GX 带 5 人，绩效/成长/分工都管），二是大厂里虚线带队——MT/XM 虽然组织上不挂人，但我是项目技术 Owner，实际负责技术方案制定、任务拆解分配、Code Review、带新人、质量卡控。包括带过外包同学，需要更细粒度的任务拆解和代码审查。"
偏虚线, 不太偏日常管理, 偏业务和需求带队这样

## 四维模型

```
技术 TL 管什么？

1. 方向 — 定什么技术路线、做什么不做什么、先做什么后做什么
2. 质量 — 怎么让代码质量"自动"维持高水平（不靠嘴说）
3. 效率 — 怎么让团队快（消除阻塞和重复劳动，不是催人）
4. 人才 — 怎么让人越来越强（走了之后团队还行不行）
```

## 维度一：方向把控

> "我决定做什么、不做什么、先做什么。"

### 做什么

- 技术选型决策（为什么选 A 不选 B，trade-off 是什么）
- 半年技术路线图规划（基建 → 效率 → 体验，分阶段）
- 技术债务评估 + 优先级排序（不是有债就还，是算 ROI）
- 和产品/业务对齐技术方向（技术服务业务，不是自嗨）

### 不做什么

- 拒绝不合理的技术方案（给出替代方案）
- 控制技术栈膨胀（不随便引入新框架/库）
- 区分"该做"和"想做"

### 经验素材

| 决策 | why | 结果 |
|------|-----|------|
| XRN 选 Metro + Hermes（不选自研打包） | 社区生态 > 自研维护成本 | 按时交付，不踩坑 |
| AI 工具链选 MCP 协议（不选自建协议） | 标准协议 + 生态扩展性 | 快速对接多个 AI Agent |
| 搭建系统先做组件物料（不先做可视化拖拽） | 物料覆盖 80% 场景，ROI 高 | 3 周上线，活动创建 0.5pd→15s |

## 维度二：质量保障

> "不靠口头约定，靠机制自动卡控。"

### 全链路质量卡控体系

```
编码时 → ESLint + Prettier（IDE 实时提示，格式不对保存自动修复）
提交时 → Git Hooks（husky + lint-staged，不过检查不让 commit）
合入时 → CI Pipeline（lint + test + build 全通过才能合入 MR）
发布时 → 灰度发布 + 线上监控 + 告警（退化自动通知）

效果：不是"叫大家写好代码"，而是"写不好代码就没法合入"
```

### 推动落地的策略

```
不是一次性全上，而是分步：
  Week 1: 先上 ESLint（只报警不阻断）→ 让团队适应
  Week 2: 开启 Prettier 自动格式化（减少争论）
  Week 3: 上 Git Hooks（提交时卡控）
  Week 4: 上 CI Pipeline（合入时卡控）
  
关键：先让人感受到好处（IDE 提示帮你发现 bug），再加约束
```

### 面对阻力怎么办

| 阻力 | 应对 |
|------|------|
| "太严了影响效率" | 用数据说话：上线后 bug 率降了 X%，实际是提效 |
| "旧代码全报错" | 增量推进：只检查新改的文件（lint-staged） |
| "不想改习惯" | 从 Prettier 开始（零认知成本，保存自动格式化） |

## 维度三：效率提升

> "不是催人快，是消除阻塞和重复劳动。"

### 提效公式

```
团队效率 = 每人有效产出时间 / 总工作时间

提效手段 = 消除三类浪费：
  1. 等待浪费：等环境、等联调、等 Review
  2. 重复浪费：重复创建项目、重复写模板代码、重复配置
  3. 认知浪费：每次都要问人、每次都要翻文档、每次都要调试
```

### 实际提效案例

| 动作 | 消除什么浪费 | 效果 |
|------|-------------|------|
| CLI 脚手架 | 重复创建项目 + 配置 | 新项目 10s 初始化 |
| 搭建系统 | 重复开发活动页 | 活动创建 0.5pd → 15s |
| 物料调试中心 | 调试等待/环境配置 | 调试链路 1min47s → 10s |
| AI 全链路（MCP） | 人工切换工具 + 重复操作 | 需求→代码→Review 自动串联 |
| CI/CD 自动化 | 手动打包/部署 | push tag → 自动发版 |

### 关键原则

- 每个提效都有**量化数据**（前后对比）
- 优先爆破**高频 × 高耗时**的节点
- 工具化 > 规范化（规范靠人遵守，工具不给你选择）

## 维度四：人才成长

> "团队能力 = 我走了之后团队还行不行。"

### 具体做法

| 机制 | 做什么 | 目的 |
|------|--------|------|
| Code Review | 不只挑错，附带"为什么这样写更好" | 教学式 Review |
| 新人导师制 | 入职 2 周有人带，1 个月有独立产出 | 快速上手 |
| 技术分享 | 每两周一次，轮值分享 | 知识扩展 + 表达能力 |
| 文档驱动 | 关键决策写 ADR（Architecture Decision Record） | 知识不在个人脑子里 |
| 渐进授权 | 从小模块 → 子系统 → 独立负责方向 | 让人成长为能独当一面 |

### 成长路径设计

```
初级 → 中级 → 高级 的能力台阶：

初级：能按要求完成功能开发
中级：能独立做技术方案 + 做技术决策
高级：能定方向 + 带人 + 建体系

TL 的工作 = 帮每个人看清自己在哪个台阶 + 给对应的成长机会
```

## 面试表达要点

### 不要讲

- 我管 5 个人
- 我负责分配任务和排期
- 我做周报和 1on1
- 我负责跨团队沟通

### 要讲

- 我建了 X 体系，解决了 Y 问题，效果是 Z（量化）
- 我推动了 X 落地，遇到了 Y 阻力，我怎么解决的
- 我做的 X 被 N 个团队复用 / 影响了 M 个人
- 我不在了之后这个体系还在跑

### 核心叙事线

> "我带团队不是排活，是建体系。质量靠 CI 卡控不靠嘴说，效率靠工具提升不靠加班，方向靠技术判断不靠上面安排。我走了之后这些体系还在跑。"

## STAR 故事素材

### Story 1：质量治理推动

```
S: 团队无代码规范，bug 率高，Review 流于形式
T: 建立全链路质量卡控机制
A: 分步推进（ESLint → Hooks → CI），先让人感受好处再加约束
R: bug 率下降 X%，Review 通过率提升，新人上手快
```

### Story 2：研发资产沉淀

```
S: 每次活动开发都从零写，重复劳动多
T: 建立可复用的研发资产体系
A: 搭建系统（CLI + 物料 + 可视化配置）
R: 活动创建 0.5pd → 15s，服务 N 个业务方向
```

### Story 3：AI 体系规划与落地

```
S: 团队用 AI 停留在"问 ChatGPT"层面，没有系统性融入研发流程
T: 规划并落地 AI 开发体系
A: 选 MCP 协议串联全链路（需求→设计→编码→测试→Review→Bug）
R: 开发效率可量化提升，形成可推广的方法论
```

### Story 4：跨团队协作

```
S: 上下游对接成本高（每次都要开会对齐、接口文档不清晰）
T: 降低协作成本
A: 标准化接口契约（GraphQL Schema / OpenAPI）+ 文档自动生成 + 工具链支持
R: 对接成本从"开会 → 开发 → 联调"变成"看文档 → 自助对接"
```

## 感悟：工程化建设 = 管理

> Tech Leader 的管理不是"排活 + 周报"，是**通过工程手段实现团队效率和质量目标**。
>
> 本质：用系统/工具/机制替代人肉管理。不是"盯着人干活"，是"造系统让人不出错/更高效"。

**我做的每一项工程化建设，都是管理叙事的素材**：

| 我做的事 | 管理视角怎么讲 |
|---------|---------------|
| Lint → Git Hooks → CI/CD 全链路卡控 | 建立质量门禁机制，从人治到制度保障 |
| CLI 脚手架 + 调试效率 10x | 识别研发瓶颈 → 工具爆破 → 团队整体提效 |
| 搭建系统服务多业务方向 | 沉淀可复用基建，影响力从团队扩展到部门 |
| Code Review 机制 | 建立技术规范 + 知识传递机制 |
| 新人 onboarding | 人才梯队建设，缩短新人产出周期 |
| AI 工具链推广 | 技术战略落地 — 从个人实践到团队标准 |
| 可观测体系建设 | 让系统状态可见 — 数据驱动决策，不靠感觉 |

**核心认知**：人效治理、质量治理、工程化建设 — 这些本身就是 Tech Leader 的管理内容。不需要"带了多少人"才能讲管理，**通过机制和工具让团队变好 = 管理**。

## 管理叙事升级：三层递进

> DDD + 质量 + 人效 → 全链路治理 + 推动团队 AI Native 化 + 推动业务 AI 化

### 叙事框架

```
Layer 1: 全链路质量人效治理 — 证明你能"建体系"
Layer 2: 推动团队 AI Native 化 — 证明你能"引领变革"
Layer 3: 推动业务 AI 化 — 证明你能"创造业务价值"
```

### Layer 1：全链路治理（DDD + 质量 + 人效）

> "我通过工程机制系统性治理质量和人效"

| 维度 | 手段 | 量化结果 |
|------|------|---------|
| 质量治理 | Lint → Hooks → CI 全链路卡控 | Bug 率下降、Review 效率提升 |
| 人效治理 | CLI / 搭建系统 / 调试工具 10x 提效 | 活动创建 0.5pd→15s |
| DDD 领域治理 | 模块划分、边界守卫、依赖方向约束 | 改动收敛、跨模块 Bug 减少 |
| 可观测治理 | OTel + 告警 + 异常分析 | 故障定位 min 级 → sec 级 |

### Layer 2：推动团队 AI Native 化

> "我不只是自己用 AI，是让整个团队的研发流程原生集成 AI"

| 阶段 | 做了什么 | 效果 |
|------|---------|------|
| 工具选型 | 选定 MCP 协议 + Agent 工具链 | 标准化，不各玩各的 |
| 流程嵌入 | AI Code Review / AI Test Gen / AI Doc | 融入 CI，不靠人手动触发 |
| 团队推广 | Prompt 模板 + 最佳实践 + 培训 | 从"问 ChatGPT"→ 系统化使用 |
| 度量 | AI 辅助代码占比 / 缺陷率对比 | 数据证明 ROI |

### Layer 3：推动业务 AI 化

> "技术不止于效率，能直接创造业务价值"

| 场景 | 做法 | 业务价值 |
|------|------|---------|
| 智能搭建 | LowCode + AI 自动生成页面 | 运营自助率↑，开发需求量↓ |
| 智能客服/FAQ | RAG + 知识库 | 人工介入率↓ |
| 智能推荐/AB | AI 驱动实验平台 | 转化率可量化提升 |

### 面试一句话

> "我的管理路径是：先建全链路质量人效体系（让团队不出错、跑得快），再推动 AI Native 化（让工具链原生集成 AI），最终推动业务 AI 化（让技术直接产出业务价值）。三层递进，从守到攻。"

## 回答策略：问点 → 答面 → 答体系

> 每个问题都是一个入口，不是回答完就结束，是借这个点把你的体系拉出来。

**核心心法**：抓住每一次回答的机会，都是证明自己价值的机会。用体系的线去串起知识和经验叙事。

```
面试官问什么    →  你回答什么                           →  拉出什么体系
─────────────────────────────────────────────────────────────────────
代码质量        →  全链路卡控（Lint→CI→告警）          →  质量人效治理体系
Node 性能       →  P95/5xx/EL Lag + OTel 采集         →  全端可观测体系
微前端          →  qiankun vs MF 选型判断              →  架构决策力 + 落地经验
工程化          →  CLI/搭建/调试 10x                   →  系统性提效思维
AI 相关         →  SDD/MCP/Harness                    →  三层递进（工具→团队→业务）
带团队/管理     →  工程化建设 = 管理                    →  通过机制让团队自运转
```

**心智模型**：面试不是 Q&A，是**借题发挥**。对方问一个点，你给一个面，让他感受到"这人背后有一套完整的思考体系"。
