# 入职展开工作的方法论

> 解决什么问题：被问"你入职后打算怎么开展工作？"怎么回答。
> 本质：收集问题 → 分优先级 → 分短期/长期方案 → 给出实施计划 → AI 加速执行。
> 核心观点：AI 时代节奏可以更快，呈现量级提升。同样的事情，以前要 3 个月，现在 1 个月搞定。

## 方法论框架

```
入职第一件事不是写代码，是收集问题。

1. 收集问题 — 了解现状（技术/业务/团队/流程）
2. 分优先级 — 按影响面 × 紧急度排序
3. 分短期/长期 — 快赢（1 周见效）+ 中期（1 月）+ 长期（3 月）
4. 给出实施方案 — 每个问题有技术方案 + 量化目标
5. AI 加速执行 — 传统 3 个月干的事，AI 辅助 1 个月搞定
```

## 第一周：收集问题

### 怎么收集

| 渠道 | 了解什么 |
|------|---------|
| 读代码 | 架构现状、技术债、代码质量 |
| 读文档 | 已有规范/决策记录（如果有的话） |
| 和团队聊 | 痛点（构建慢？联调难？发布流程繁琐？） |
| 和产品聊 | 业务方向、近期规划、对前端的期望 |
| 跑一遍流程 | 从 clone 到本地跑起来到发布，体验全链路 |
| 看监控 | 线上错误率、性能指标、崩溃率 |

### 典型问题清单

```
技术维度：
  □ 构建速度（几分钟？）
  □ 代码规范（有 lint 吗？有 CI 卡控吗？）
  □ 测试覆盖（有单测吗？覆盖率？）
  □ 性能（首屏时间？白屏率？）
  □ 技术栈版本（有过时的依赖吗？）
  □ 架构健康度（模块耦合？循环依赖？）

流程维度：
  □ 发布流程（手动？自动？耗时？）
  □ Code Review（有机制吗？流于形式？）
  □ 需求到上线全链路（瓶颈在哪？）

团队维度：
  □ 团队规模和分工
  □ 新人上手成本
  □ 文档体系（有没有？是否更新？）
```

## 第二周：分析 + 定方案

### 优先级矩阵

```
              影响面大
                │
    ┌───────────┼───────────┐
    │  重要紧急  │  重要不紧急 │
    │  (快赢)   │  (中长期)  │
    │  立即做    │  规划做    │
────┼───────────┼───────────┼────
    │  不重要紧急│  不重要不紧急│
    │  (委托/快 │  (不做)    │
    │   速解决) │           │
    └───────────┼───────────┘
                │
              影响面小
```

### 分层方案模板

| 层 | 时间 | 做什么 | 目标 |
|---|---|---|---|
| 快赢 | 1-2 周 | ESLint + Prettier 接入、明显 bug 修复 | 立刻见到改善 |
| 中期 | 1-2 月 | CI/CD 搭建、性能优化首屏、组件库启动 | 质量可度量提升 |
| 长期 | 3-6 月 | 架构升级（微前端/技术栈迁移）、AI 体系建设 | 体系化 |

## 第三周起：分阶段执行

### 执行节奏

```
Week 1: 了解 + 融入（不急着改）
Week 2: 方案输出 + 对齐（和 leader/团队达成共识）
Week 3-4: 快赢落地（ESLint/CI/明显优化）→ 建立信任
Month 2: 中期专项推进（性能/组件库/工程化）
Month 3+: 长期架构建设（微前端/AI 体系）

关键：先做快赢建立信任，再推大事。不要一上来就推架构重构。
```

## AI 加速：量级提升

> AI 时代，同样的技术建设工作节奏可以快一个量级。

### 传统 vs AI 辅助

| 任务 | 传统 | AI 辅助 |
|------|------|---------|
| 组件库搭建 | 2-3 月 | 2-3 周（AI 生成组件骨架 + 文档） |
| ESLint 规则定制 | 1 周 | 1 天（AI 根据代码风格生成规则集） |
| 单测补全 | 持续投入 | AI 自动生成测试用例，人工 review |
| 技术方案文档 | 2-3 天 | 半天（AI 辅助写 + 人工润色） |
| Code Review | 人工逐行 | AI pre-review + 人工决策 |
| 全链路质量卡控 | 搭建 CI 1-2 周 | MCP 串联工具链，半天搞定 |

### 怎么讲

> "AI 时代不只是'写代码快了'，是整个技术建设的节奏可以提一个量级。以前搭一套 CI/CD + 组件库 + 文档体系要 3 个月，现在 AI 辅助生成骨架 + 人工把关质量，1 个月就能出完整体系。我的方法论是：AI 负责量产，人负责决策和质量。"

## 叙述参考

### 被问"入职后怎么开展工作"

> "第一周我不写代码，先收集问题——读代码了解架构现状、和团队聊痛点、跑一遍全链路体验。第二周输出方案，按影响面和紧急度分优先级，快赢先做（比如 lint/CI）建立信任。然后分中期和长期推进。
>
> AI 时代的优势是节奏可以更快——组件库、文档、测试这些'量大但重复'的工作，AI 可以量产骨架，人把关质量。传统要 3 个月的技术建设，现在 1 个月可以出成果。"

### 被问"你怎么推动一件事落地"

> "分三步：先量化问题（现在多差），再给方案（怎么变好），最后给数据（好了多少）。每一步都有交付物——不是口头说'我要优化性能'，而是'首屏从 3s 到 1s，方案是 X，预计 2 周'。"
