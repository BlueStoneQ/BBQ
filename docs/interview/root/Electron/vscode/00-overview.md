**IDE = 文本编辑器 + 语言智能 + 构建调试 + 扩展生态**

VSCode 的突破在于：用 Web 技术栈实现了传统 Native IDE 的体验，同时用插件化架构获得了无限扩展性。

### 视角三: 数据流 (一次代码补全的完整路径)

```
用户输入 "."
    │
    ▼
[DOM Event] → KeyboardEvent
    │
    ▼
[Monaco InputHandler] → 判断非快捷键，写入文本模型
    │
    ▼
[PieceTree] → 插入 "."，更新行信息
    │
    ▼
[CompletionTrigger] → "." 是 triggerCharacter → 触发补全
    │
    ▼
[LanguageFeatureRegistry] → 找到 Python CompletionProvider
    │
    ▼
[RPC Protocol] ═══ MessagePort ═══► [Extension Host]
    │                                     │
    │                                     ▼
    │                              [ExtHostLanguageFeatures]
    │                                     │
    │                                     ▼
    │                              [Language Client] ─── stdio ──► [pylsp Server]
    │                                     │                              │
    │                                     │         JSON-RPC             │
    │                                     │    textDocument/completion   │
    │                                     │◄─────────────────────────────┘
    │                                     │
    │◄═══ CompletionList (RPC返回) ═══════╡
    │
    ▼
[SuggestWidget] → 渲染补全列表 UI
    │
    ▼
[用户选择] → 应用补全文本 → PieceTree 更新 → 视图重绘
```

## 关键洞察

1. **分层是第一原则** — base → platform → editor → workbench，严格单向依赖
2. **进程隔离是安全基石** — 插件永远运行在独立进程，崩溃不影响主界面
3. **协议优于实现** — IPC Protocol、LSP、DAP 都是先定义协议再做实现
4. **DI 是胶水** — 服务注册/发现全靠依赖注入，测试友好
5. **环境标签是跨平台密码** — `common/browser/node/electron-main` 目录约定实现代码隔离
