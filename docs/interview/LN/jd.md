# LN — App 前端开发（AR 眼镜 + 技能编排）

> 产品：AR 智能眼镜的技能编排系统（手机 App + 眼镜端渲染）
>
> 定位：前端主力，手机 App 开发为主战场 + 眼镜端前端渲染
>
> 技术栈：Flutter（跨端）+ Android/iOS 原生 + Server-Driven UI

---

## 核心职责拆解

### 1. 手机端 App 开发（主战场）

| 模块 | 具体内容 |
|------|---------|
| 技能流程看板 | 卡片流 / 流程树可视化 |
| 语音微调交互 | 语音相关 UI 交互 |
| 版本回溯 UI | 版本历史 / diff 展示 |
| 技能卡片创建端 | 技能选择 → 对话编辑 → 眼镜端模拟预览渲染 → 模版管理 |
| 技能商店 App 端 | 浏览 / 筛选 / 订阅 / 开发者上架 |
| 原生差异处理 | 地图选点器、时间选择轮等原生控件调起 |

### 2. 眼镜端 / 嵌入式前端渲染

- 根据卡片 JSON/XML 协议，在眼镜端受限硬件环境中实现**高性能渲染**
- 局部刷新（数据级）与整体样式更新（状态级）
- 参与制定跨硬件规格的前端 UI 协议规范
- 确保技能链路在不同设备间可复用，只需适配渲染层

### 3. 前后端协议联调

- 与后端/AI 侧对接 **Skill DSL** 和 **Preview JSON** 双层数据协议
- 保障编排流程数据到前端分块卡片的精准映射
- 推动前端渲染层的规范化，减少三方技能接入的介入成本

### 4. Demo 快速交付

- 早期以产品 Demo 体验为核心目标
- 在需求尚未完全确定的情况下，做出有说服力的交互原型和可运行 Demo
