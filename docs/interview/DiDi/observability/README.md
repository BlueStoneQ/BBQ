## 可观测闭环模型

> 可观测不是"看看数据"，是建立"问题自动浮现 → 自动通知 → 快速定位 → 治理验证"的闭环。
>
> 核心价值不在采集，在于**缩短 MTTR**（Mean Time To Recovery，故障恢复时间）。

```
采集 → 分析 → 告警 → 治理 → (验证) → 采集...
 ↑                                      ↓
 └──────── 闭环：修复后指标回落 ──────────┘
```

| 阶段 | 做什么 | 关键能力 |
|------|--------|---------|
| **1. 采集（Collect）** | OTel 探针 / 埋点 → 拿到原始数据 | 覆盖率：哪些指标在采、哪些没覆盖 |
| **2. 分析（Analyze）** | 指标聚合 + 异常定位（哪个接口/页面/代码） | 下钻能力：从宏观到微观 |
| **3. 告警（Alert）** | 阈值触发 → 通知到人（不靠人盯大盘） | 告警质量：不漏报、不误报 |
| **4. 治理（Action）** | 定位根因 → 修复 → 上线 → 验证指标回落 | 闭环速度：MTTR 多短 |

**缺任何一环都不完整**：
- 只有采集没有告警 = 被动看数据，出事了才知道
- 只有告警没有分析 = 知道出事了但不知道哪里出事
- 只有分析没有治理 = 知道问题在哪但没人修
- 只有治理没有验证 = 修了但不知道修没修好
