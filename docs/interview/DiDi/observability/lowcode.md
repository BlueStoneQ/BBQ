# 搭建页面可观测

> 解决什么：低代码/搭建出来的页面，怎么保证性能下限？怎么知道有没有问题？

---

## 核心指标

| 指标 | 含义 | 阈值 | 采集 |
|------|------|------|------|
| **渲染成功率** | 页面能不能正常渲出来 | > 99%，< 98% 告警 | ErrorBoundary catch + 上报 |
| **白屏率** | 渲染了但什么都没有 | < 1%，> 3% 告警 | 白屏检测 SDK（DOM 采样） |
| **组件数** | 单页面组件总量 | < 50 正常，> 100 告警 | schema 分析 |
| **嵌套深度** | 容器嵌套层数 | < 5 正常，> 8 告警 | schema 递归计算 |

---

## 保障手段（前置 + 运行时 + 兜底）

| 阶段 | 手段 | 做什么 |
|------|------|--------|
| **编辑时** | 实时提示 | 嵌套超 5 层 → 编辑器标红提示 |
| **保存时** | schema 校验 | 组件数/嵌套深度/图片大小 超限 → 拒绝保存 |
| **发布时** | 性能预检（门禁） | 模拟渲染 → 渲染时间 > 3s 不让发布 |
| **运行时** | ErrorBoundary | 单组件崩溃 → 降级展示，不影响整页 |
| **运行时** | 渲染超时 | 单组件渲染 > 2s → 超时降级 |
| **兜底** | 全局白屏检测 | 整页白屏 → 展示兜底 UI + 上报 |

---

## 采集方案

不需要 OTel（搭建页面跑在浏览器里），用自建 SDK：

```ts
// 渲染成功率
try {
  renderPage(schema)
  report({ type: 'render_success' })
} catch (e) {
  report({ type: 'render_fail', error: e.message })
  showFallbackUI()
}

// 白屏检测（DOM 采样）
setTimeout(() => {
  const hasContent = document.querySelector('#app')?.children.length > 0
  if (!hasContent) report({ type: 'blank_screen' })
}, 3000)
```

## 异常排查

| 异常 | 原因 | 处理 |
|------|------|------|
| 渲染失败 | schema 格式错误 / 组件代码 Bug | Sentry 堆栈 → 修组件 |
| 白屏 | JS 加载失败 / 无限循环渲染 | 白屏检测 → 兜底页 + 告警 |
| 渲染慢 | 嵌套过深 / 组件过多 / 大图 | 性能预检拦住 → 提示优化 |
