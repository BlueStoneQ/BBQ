## 一、项目背景与价值

### 1.1 业务痛点
- **开发效率低**：每次营销活动都需要从零开发，周期长（2-3周）
- **人力成本高**：大促期间需要大量开发人员支持活动页面开发
- **质量不稳定**：重复开发导致代码质量参差不齐，bug率高
- **维护困难**：活动页面代码分散，难以统一维护和优化

### 1.2 解决方案
通过低代码搭建平台，实现营销活动页面的可视化搭建，将开发周期从2-3周缩短到1-2天。

### 1.3 核心价值
- **效率提升**：页面开发效率提升 10 倍以上
- **成本降低**：减少 80% 的重复开发工作
- **质量保障**：统一的组件库和规范，保证代码质量
- **业务赋能**：运营人员可自主搭建简单页面，释放开发资源

```typescript
// 简化版 JS 解释器
class JSInterpreter {
  private context: any = {};
  
  execute(code: string) {
    // 1. 解析代码为 AST
    const ast = parse(code);
    
    // 2. 遍历 AST 执行
    this.executeNode(ast);
  }
  
  private executeNode(node: ASTNode) {
    switch (node.type) {
      case 'VariableDeclaration':
        this.executeVariableDeclaration(node);
        break;
      case 'FunctionDeclaration':
        this.executeFunctionDeclaration(node);
        break;
      case 'CallExpression':
        return this.executeCallExpression(node);
      // ... 其他节点类型
    }
  }
}
```

#### 3.4.3 组件通信方案

**设计思路**：基于 EventBus 实现组件间通信

```typescript
// EventBus 实现
class EventBus {
  private events: Map<string, Function[]> = new Map();
  
  on(eventName: string, handler: Function) {
    if (!this.events.has(eventName)) {
      this.events.set(eventName, []);
    }
    this.events.get(eventName).push(handler);
  }
  
  emit(eventName: string, ...args: any[]) {
    const handlers = this.events.get(eventName) || [];
    handlers.forEach(handler => handler(...args));
  }
}
```

**组件配置**：

```json
{
  "broadcast": [{
    "eventName": "clickSwitch",
    "label": "点击了 Tab 切换"
  }],
  "broadcastReceiver": [{
    "eventName": "changeColor",
    "label": "改变颜色"
  }]
}
```

**使用示例**：

```javascript
// 组件 A：广播事件
Component({
  methods: {
    handleClick() {
      eventBus.emit('clickSwitch');
    }
  }
});

// 组件 B：监听事件
Component({
  onLoad() {
    eventBus.on('clickSwitch', () => {
      eventBus.emit('changeColor');
    });
    
    eventBus.on('changeColor', () => {
      this.changeColor();
    });
  }
});
```

## 五、技术难点与解决方案

### 5.1 小程序动态化限制

**难点**：小程序不支持 `eval`、`new Function` 等动态执行代码的 API

**解决方案**：
- 自研 JS 解释器，通过解析 AST 执行代码
- 将 JS 代码转换为可执行的数据结构
- 在沙箱环境中安全执行

### 5.2 组件依赖管理

**难点**：单包单组件如何处理依赖关系？

**解决方案**：
- **JS 依赖**：通过 Babel 分析 import 语句，生成依赖图，打包时内联依赖代码
- **组件依赖**：将依赖组件的 XML 解析为 JSON Schema，打平后合并到当前组件的 children 中
- **External 配置**：排除运行时已有的依赖（如 React、Vue 等）

### 5.3 组件通信

**难点**：搭建的页面中，组件间如何通信？

**解决方案**：
- 基于 EventBus 实现发布-订阅模式
- 在搭建端配置 broadcast 和 broadcastReceiver
- 运行时自动注册事件监听

### 5.4 性能优化

**难点**：大型页面渲染性能问题

**解决方案**：
- **虚拟滚动**：长列表使用虚拟滚动技术
- **懒加载**：组件按需加载
- **缓存策略**：JSON Schema 缓存
- **预加载**：结合 PreFetcher 预请求数据

### 8.2 简化版 EventBus

```typescript
class EventBus {
  private events = new Map<string, Function[]>();
  
  on(event: string, handler: Function) {
    if (!this.events.has(event)) {
      this.events.set(event, []);
    }
    this.events.get(event)!.push(handler);
  }
  
  emit(event: string, ...args: any[]) {
    const handlers = this.events.get(event) || [];
    handlers.forEach(handler => handler(...args));
  }
  
  off(event: string, handler?: Function) {
    if (!handler) {
      this.events.delete(event);
    } else {
      const handlers = this.events.get(event) || [];
      const index = handlers.indexOf(handler);
      if (index > -1) {
        handlers.splice(index, 1);
      }
    }
  }
}
```
