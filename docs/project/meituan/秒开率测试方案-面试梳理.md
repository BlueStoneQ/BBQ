## 一、项目背景与价值

### 1.1 业务痛点
- **性能无法量化**：无法准确衡量小程序页面的加载性能
- **用户体验难评估**：不知道用户实际感知的页面加载速度
- **优化无方向**：缺乏数据支撑，不知道从哪里优化
- **问题难定位**：线上性能问题无法及时发现和定位

### 1.2 核心概念

**秒开率（FST - First Screen Time）**：
- 定义：首屏渲染时间 < 1 秒的页面占比
- 计算公式：秒开率 = (首屏时间 < 1s 的 PV / 总 PV) × 100%
- 意义：衡量页面加载性能的核心指标

**首屏时间**：
- 定义：从页面开始加载到首屏内容完全呈现的时间
- 起点：小程序 Page.onLoad 触发时刻
- 终点：首屏内容渲染完成并稳定的时刻

### 1.3 解决方案
通过自研性能监控 SDK，自动采集页面首屏时间，实现性能数据的可视化和监控告警。

### 1.4 核心价值
- **性能可视化**：实时监控页面性能数据
- **问题快速定位**：及时发现性能劣化
- **优化有方向**：基于数据驱动的性能优化
- **用户体验提升**：通过性能优化提升用户满意度

## 二、技术方案选型

### 2.1 首屏时间度量模型对比

#### 方案一：可见元素增幅稳定模型

**原理**：计算可见元素增幅趋于稳定的时刻作为首屏时间

**优点**：
- 适用于所有布局类型
- 准确度高

**缺点**：
- 强依赖 DOM Mutation API（小程序不支持）
- 需要处理频繁变动元素（如跑马灯）

**适用场景**：Web 端

#### 方案二：可见元素填充率模型

**原理**：计算页面可见元素在横轴和纵轴上的投影比率，超过阈值即认为首屏完成

**优点**：
- 适用于所有布局类型
- 理论上准确度高

**缺点**：
- 需要遍历所有可见元素，性能开销大
- 小程序逻辑层和渲染层分离，查询元素信息需要跨层通信，性能损耗不可控

**适用场景**：Native App

#### 方案三：可见元素触底模型（最终选型）

**原理**：在页面底部插入靶点元素，计算靶点元素触底时间作为首屏时间

**优点**：
- 性能开销小，只需查询单个元素
- 实现简单，易于维护
- 适用于流式布局（大部分小程序页面）

**缺点**：
- 需要手动插入靶点元素（侵入性）
- 需要处理页面不足一屏的边界情况

**适用场景**：小程序

### 2.2 最终方案设计

**核心思路**：
1. 在页面实际内容容器底部插入零高度靶点元素
2. 实时监测靶点元素位置
3. 当靶点元素触底或页面稳定时，计算首屏时间

**边界处理**：
- **页面超过一屏**：靶点触底时刻即为首屏时间
- **页面不足一屏**：回溯计算靶点首次到达稳定位置的时刻

**兜底方案**：
- 用户首次交互前的最后一次视图变动时刻

## 三、系统架构设计

### 3.1 整体架构

```
┌─────────────────────────────────────────────────────┐
│                   FST SDK 架构                       │
├─────────────────────────────────────────────────────┤
│                                                       │
│  ┌──────────┐   ┌──────────┐   ┌──────────┐        │
│  │  注册模块  │──▶│  测速模块  │──▶│  上报模块  │        │
│  │ Register │   │ Sampling │   │  Report  │        │
│  └──────────┘   └──────────┘   └──────────┘        │
│       │              │               │               │
│       ▼              ▼               ▼               │
│   初始化配置      生命周期拦截    数据上报           │
│   Hook组件       采样计算         指标平台           │
│                                                       │
└─────────────────────────────────────────────────────┘
```

### 3.2 核心模块

#### 3.2.1 注册模块（Register）
- **职责**：初始化 SDK 配置，Hook 全局 Component
- **时机**：小程序 app.onLaunch 阶段
- **功能**：
  - 记录项目元信息（bundleId、环境等）
  - 获取远程配置（采样率等）
  - 重写全局 Component 方法

#### 3.2.2 测速模块（Sampling）
- **职责**：采集页面渲染数据，计算首屏时间
- **核心能力**：
  - 生命周期拦截（onLoad、onHide、onUnload）
  - setData 拦截（Page 和 Component）
  - 靶点位置采样
  - 振幅算法计算稳定时刻

#### 3.2.3 上报模块（Report）
- **职责**：上报性能数据到监控平台
- **时机**：页面 onHide 或 onUnload
- **数据**：首屏时间、页面路径、用户信息等

## 四、核心技术实现

### 4.1 生命周期拦截（AOP）

**设计思路**：通过 AOP（面向切面编程）拦截页面生命周期，注入测速逻辑

**实现方式**：

```javascript
// Base 工厂函数
const Base = function(pageObj) {
  return fstPageWrapper(pageObj);
}

// FST Page Wrapper
function fstPageWrapper(pageObj) {
  // 1. 拦截 onLoad
  const originalOnLoad = pageObj.onLoad;
  pageObj.onLoad = function(...args) {
    // 注入测速逻辑
    this.fst = new FstReport(this);
    this.fst.startRecord();
    
    // Hook setData
    const originalSetData = this.setData;
    this.setData = function(data, callback) {
      originalSetData.call(this, data, () => {
        this.fst.sampling();
        callback && callback();
      });
    };
    
    // 执行原 onLoad
    originalOnLoad && originalOnLoad.call(this, ...args);
  };
  
  // 2. 拦截 onHide
  const originalOnHide = pageObj.onHide;
  pageObj.onHide = function(...args) {
    this.fst.tryComputeFst();
    this.fst.stopRecord();
    this.fst.report();
    
    originalOnHide && originalOnHide.call(this, ...args);
  };
  
  // 3. 拦截 onUnload（兜底）
  const originalOnUnload = pageObj.onUnload;
  pageObj.onUnload = function(...args) {
    this.fst.tryComputeFst();
    this.fst.stopRecord();
    this.fst.report();
    
    originalOnUnload && originalOnUnload.call(this, ...args);
  };
  
  return pageObj;
}
```

**使用方式**：

```javascript
// 页面中使用
Page(Base({
  onLoad() {
    // 业务逻辑
  }
}));
```

### 4.2 Component Hook

**问题**：如何监听组件的 setData？

**解决方案**：在 SDK 注册时重写全局 Component 方法

```javascript
function register() {
  const originalComponent = Component;
  
  Component = function(compObj) {
    // Hook attached 生命周期
    const originalAttached = compObj.attached;
    compObj.attached = function(...args) {
      // Hook setData
      hookSetData.call(this);
      
      // 执行原 attached
      originalAttached && originalAttached.call(this, ...args);
    };
    
    // 返回原 Component 执行结果
    return originalComponent(compObj);
  };
}

function hookSetData() {
  const originalSetData = this.setData;
  
  this.setData = function(data, callback) {
    originalSetData.call(this, data, () => {
      // 获取当前页面
      const pages = getCurrentPages();
      const currentPage = pages[pages.length - 1];
      
      // 触发采样
      currentPage.fst && currentPage.fst.sampling();
      
      // 执行原 callback
      callback && callback();
    });
  };
}
```

### 4.3 采样算法（Sampling）

**核心数据结构**：

```typescript
interface Record {
  targetTop: number;    // 靶点距离页面顶部的距离
  timeStamp: number;    // 记录时间戳
}

class FstReport {
  private recordList: Record[] = [];
  private startTime: number = 0;
  private endTime: number = 0;
  private status: 'wait' | 'sampling' | 'stop' = 'wait';
  private windowHeight: number = 0;
}
```

**采样流程**：

```javascript
class FstReport {
  // 开始记录
  startRecord() {
    this.startTime = Date.now();
    this.windowHeight = wx.getSystemInfoSync().windowHeight;
    this.status = 'sampling';
  }
  
  // 采样
  async sampling() {
    if (this.status !== 'sampling') return;
    
    // 获取靶点位置
    const targetTop = await this.getTargetViewTop();
    const now = Date.now();
    
    // 记录数据
    this.recordList.push({ targetTop, timeStamp: now });
    
    // 检查是否触底
    if (targetTop >= this.windowHeight) {
      this.endTime = now;
      this.status = 'stop';
      return;
    }
    
    // 振幅算法：向上追溯稳定时刻
    this.checkStability();
  }
  
  // 获取靶点位置
  getTargetViewTop() {
    return new Promise((resolve) => {
      const query = wx.createSelectorQuery();
      query.select('.fst-target').boundingClientRect();
      query.selectViewport().scrollOffset();
      query.exec((res) => {
        // 靶点到屏幕顶部的距离 + 页面滚动距离
        const top = res[0].top + res[1].scrollTop;
        resolve(top);
      });
    });
  }
}
```

### 4.4 振幅算法（向上追溯法）

**核心思想**：通过分析靶点位置变化的振幅，找到页面稳定的时刻

**算法流程**：

```javascript
checkStability() {
  const len = this.recordList.length;
  if (len < 2) return;
  
  const lastRecord = this.recordList[len - 1];
  const prevRecord = this.recordList[len - 2];
  
  // 判断是否超过 100ms 间隔（可能稳定）
  if (lastRecord.timeStamp - prevRecord.timeStamp < 100) return;
  
  // 向上追溯，找到最早的稳定点
  for (let i = len - 1; i >= 1; i--) {
    const cur = this.recordList[i];
    const pre = this.recordList[i - 1];
    
    // 振幅大于 20px，说明还在变化
    if (cur.targetTop - pre.targetTop > 20) {
      break;
    }
    
    // 振幅小于 20px，持续向上找最早的稳定点
    this.endTime = pre.timeStamp;
    this.status = 'stop';
  }
}
```

**算法示意图**：

```
靶点位置
  ↑
  │     ●  ← 最后一次记录
  │    ●   ← 间隔 < 20px（稳定）
  │   ●    ← 间隔 < 20px（稳定）
  │  ●     ← 间隔 < 20px（稳定）- 选为 endTime
  │ ●      ← 间隔 > 20px（不稳定）
  │●
  └──────────────────────▶ 时间
```

**关键参数**：
- **时间阈值**：100ms（超过此时间可能已稳定）
- **位置阈值**：20px（振幅小于此值认为稳定）

### 4.5 计算首屏时间

```javascript
tryComputeFst() {
  if (this.status !== 'stop') return;
  
  // 计算首屏时间
  const fst = this.endTime - this.startTime;
  
  return fst;
}
```

## 五、技术难点与解决方案

### 5.1 小程序双线程架构

**难点**：小程序逻辑层和渲染层分离，无法直接访问 DOM

**解决方案**：
- 使用 `wx.createSelectorQuery()` API 查询元素位置
- 通过 `boundingClientRect()` 获取元素边界信息
- 结合 `scrollOffset()` 获取页面滚动距离

### 5.2 setData 异步问题

**难点**：setData 是异步的，如何确保在渲染完成后采样？

**解决方案**：
- 在 setData 的 callback 中进行采样
- callback 会在渲染完成后执行

### 5.3 组件 setData 监听

**难点**：如何监听所有组件的 setData？

**解决方案**：
- 在 SDK 注册时重写全局 Component
- 在 Component.attached 中 Hook setData
- 通过 getCurrentPages() 获取当前页面实例

### 5.4 页面不足一屏

**难点**：页面内容不足一屏时，靶点无法触底

**解决方案**：
- 使用振幅算法回溯计算稳定时刻
- 兜底方案：用户首次交互时刻

### 5.5 性能开销

**难点**：频繁查询 DOM 会影响性能

**解决方案**：
- 只查询单个靶点元素，而非所有元素
- 采样频率控制（每次 setData 后采样一次）
- 计算耗时约 50ms，误差在 100ms 左右

## 六、面试高频问题

### 6.1 什么是秒开率？如何定义首屏时间？

**回答要点**：
- **秒开率**：首屏时间 < 1s 的页面占比
- **首屏时间**：从页面开始加载到首屏内容完全呈现的时间
- **起点**：Page.onLoad 触发时刻
- **终点**：首屏内容渲染完成并稳定的时刻

### 6.2 为什么选择靶点触底模型？

**回答要点**：
- **性能考虑**：只需查询单个元素，性能开销小
- **实现简单**：相比其他方案更易实现和维护
- **适用场景**：适合小程序的流式布局
- **权衡取舍**：虽有侵入性，但性能和准确度的平衡最优

### 6.3 如何处理页面不足一屏的情况？

**回答要点**：
- **振幅算法**：通过分析靶点位置变化，找到稳定时刻
- **向上追溯**：从最后一次记录向前找，找到最早的稳定点
- **稳定判定**：连续记录间隔 < 20px 且时间间隔 > 100ms
- **兜底方案**：用户首次交互时刻

### 6.4 如何实现生命周期拦截？

**回答要点**：
- **AOP 思想**：面向切面编程，在不修改原代码的情况下注入逻辑
- **实现方式**：通过 Base 工厂函数包装 pageObj
- **拦截时机**：onLoad、onHide、onUnload
- **注入逻辑**：测速启动、数据采样、结果上报

### 6.5 如何监听组件的 setData？

**回答要点**：
- **重写 Component**：在 SDK 注册时重写全局 Component 方法
- **Hook attached**：在 attached 生命周期中 Hook setData
- **获取页面实例**：通过 getCurrentPages() 获取当前页面
- **触发采样**：在 setData callback 中调用页面的采样方法

### 6.6 振幅算法的原理是什么？

**回答要点**：
- **核心思想**：通过分析靶点位置变化的振幅判断页面是否稳定
- **时间阈值**：100ms，超过此时间可能已稳定
- **位置阈值**：20px，振幅小于此值认为稳定
- **向上追溯**：从最后一次记录向前找最早的稳定点
- **减少误差**：避免将短暂的停顿误判为稳定

### 6.7 SDK 的性能开销如何？

**回答要点**：
- **查询开销**：只查询单个靶点元素，开销小
- **采样频率**：每次 setData 后采样一次，频率可控
- **计算耗时**：约 50ms
- **误差范围**：100ms 左右
- **优化手段**：采样率控制、异步计算

```javascript
// 注册 SDK
function register({ meta, options }) {
  // 初始化配置
  FSTConfig.init(meta, options);
  
  // Hook Component
  const originalComponent = Component;
  Component = function(compObj) {
    const originalAttached = compObj.attached;
    compObj.attached = function(...args) {
      // Hook setData
      const originalSetData = this.setData;
      this.setData = function(data, callback) {
        originalSetData.call(this, data, () => {
          const pages = getCurrentPages();
          const page = pages[pages.length - 1];
          page.fst && page.fst.sampling();
          callback && callback();
        });
      };
      
      originalAttached && originalAttached.call(this, ...args);
    };
    
    return originalComponent(compObj);
  };
}

// Base 工厂函数
function Base(pageObj) {
  const originalOnLoad = pageObj.onLoad;
  pageObj.onLoad = function(...args) {
    this.fst = new FstReport(this);
    this.fst.startRecord();
    
    const originalSetData = this.setData;
    this.setData = function(data, callback) {
      originalSetData.call(this, data, () => {
        this.fst.sampling();
        callback && callback();
      });
    };
    
    originalOnLoad && originalOnLoad.call(this, ...args);
  };
  
  const originalOnHide = pageObj.onHide;
  pageObj.onHide = function(...args) {
    this.fst.stopRecord();
    this.fst.report();
    originalOnHide && originalOnHide.call(this, ...args);
  };
  
  return pageObj;
}

// FST Report 类
class FstReport {
  constructor(page) {
    this.page = page;
    this.recordList = [];
    this.startTime = 0;
    this.endTime = 0;
    this.status = 'wait';
    this.windowHeight = 0;
  }
  
  startRecord() {
    this.startTime = Date.now();
    this.windowHeight = wx.getSystemInfoSync().windowHeight;
    this.status = 'sampling';
  }
  
  async sampling() {
    if (this.status !== 'sampling') return;
    
    const targetTop = await this.getTargetViewTop();
    const now = Date.now();
    
    this.recordList.push({ targetTop, timeStamp: now });
    
    if (targetTop >= this.windowHeight) {
      this.endTime = now;
      this.status = 'stop';
      return;
    }
    
    this.checkStability();
  }
  
  checkStability() {
    const len = this.recordList.length;
    if (len < 2) return;
    
    const last = this.recordList[len - 1];
    const prev = this.recordList[len - 2];
    
    if (last.timeStamp - prev.timeStamp < 100) return;
    
    for (let i = len - 1; i >= 1; i--) {
      const cur = this.recordList[i];
      const pre = this.recordList[i - 1];
      
      if (cur.targetTop - pre.targetTop > 20) break;
      
      this.endTime = pre.timeStamp;
      this.status = 'stop';
    }
  }
  
  getTargetViewTop() {
    return new Promise((resolve) => {
      const query = wx.createSelectorQuery();
      query.select('.fst-target').boundingClientRect();
      query.selectViewport().scrollOffset();
      query.exec((res) => {
        resolve(res[0].top + res[1].scrollTop);
      });
    });
  }
  
  stopRecord() {
    this.status = 'stop';
  }
  
  report() {
    if (!this.endTime) return;
    
    const fst = this.endTime - this.startTime;
    const img = new Image();
    img.src = `https://report.com/log.gif?fst=${fst}`;
  }
}
```

## 九、扩展思考

### 9.1 如何支持多靶点？
- 在不同位置插入多个靶点
- 分别计算首屏、次屏时间
- 更精细的性能分析

### 9.2 如何自动插入靶点？
- 通过 Babel 插件在编译时自动插入
- 分析页面结构，智能选择插入位置
- 减少接入成本

### 9.3 如何排除骨架屏的干扰？
- 识别骨架屏元素
- 等待真实内容渲染完成
- 更准确的首屏时间

### 9.4 如何支持 Web 端？
- 使用 MutationObserver 监听 DOM 变化
- 采用可见元素增幅稳定模型
- 统一的 SDK 接口
