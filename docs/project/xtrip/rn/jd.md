**JD4-A**：标准 RN 业务开发，IoT 场景（蓝牙/WiFi 设备通信、设备控制 UI）。级别偏中级。

**JD4-B**：RN 方向的技术负责人/架构师，负责整个移动端技术体系。级别偏高级/专家。

### 核心竞争力

1. **你做过的事比 JD 要求的更难** — 快应用框架是"从零造一个类 RN 框架"，而 JD 只要求"用 RN + 做架构升级"。降维打击。
2. **性能优化全覆盖** — 启动（DEX 布局）、内存（PSS）、包体（153→60MB）、渲染（秒开率），每个都有数据。
3. **原生深度远超一般 RN 开发** — V8/J2V8/JNI/NDK/R8，这是 RN 架构师级别才需要的深度。
4. **工程化体系完整** — 从 CLI 到 CI/CD 到自动化测试到发布体系，全链路。
5. **当前就在 IoT 研发部** — 天然的领域匹配。

### 沟通要点

1. **主线叙事**：我在 IoT 研发部做的快应用框架，本质就是一个"类 RN 的跨端运行时"，从框架设计到性能优化到工程化全链路都有深度实践
2. **降维表达**：不只是"用 RN 开发"，而是"造过类似 RN 的框架"，对 RN 底层原理（Bridge/渲染/原生通信）有框架作者级别的理解
3. **IoT 场景**：虽然主要做框架层，但在 IoT 部门工作，理解设备交互场景和约束（性能敏感、内存受限、多端适配）
4. **性能故事**：每个优化都有数据，可以展开讲方法论

### 风险点

1. **学历要求**：明确写了"全日制本科 + 学信网验证"，确认你的学历满足
2. **IoT 业务深度**：如果被追问蓝牙协议/设备配网细节，需要提前了解基本概念
3. **JD4-A 级别偏低**：如果只有 A 的 HC，可能薪资和级别不匹配你的经验。建议直接沟通 B 的岗位
