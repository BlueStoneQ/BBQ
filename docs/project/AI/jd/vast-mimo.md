## 需求拆解

### 一、岗位本质

这不是一个传统的前后端开发岗，而是一个 **"用工程能力服务于模型训练"** 的角色。核心目标是：让大模型学会写出真实可用的工程代码。

### 二、核心能力维度拆解

#### 1. 前后端工程能力（基础盘）

- **深度**：不是写页面/接口，而是要理解企业级系统为什么复杂
- **广度**：前端（React/Vue/Next.js）+ 后端（Node.js/Python Web 框架）+ 全链路
- **关键词**：组件设计、状态管理、权限、API 设计、数据库交互、构建工具、工程化

#### 2. 测试与评测能力（核心竞争力）

- 设计可自动验证的 benchmark
- 单元测试、接口测试、E2E 测试、视觉回归测试
- 把"代码好不好"变成可量化、可自动判断的标准
- 工具：Jest/Vitest/Playwright/Cypress/Pytest

#### 3. Agent 框架与工具调用（技术方向）

- 开发 Agent 框架、工具调用框架
- 任务执行环境设计
- 熟悉 LangChain/AutoGen/OpenAI Agents SDK/CrewAI/MCP 等
- 理解 Agent 的 loop：规划 → 工具调用 → 状态反馈 → 迭代

#### 4. RL 训练环境构建（进阶方向）

- 任务定义与状态反馈
- 奖励设计（reward shaping）
- 执行沙箱
- 工具链集成与自动化验证
- 不需要你训练模型，但需要你构建模型训练的"健身房"

#### 5. 问题拆解与任务设计能力（软实力）

- 把模糊的业务场景 → 清晰的工程任务
- 把工程任务 → 可训练的样本/可评测的标准
- 分析模型失败案例 → 归因 → 设计改进方案
