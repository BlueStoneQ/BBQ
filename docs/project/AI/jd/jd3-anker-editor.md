## 需求拆解

### 一、岗位本质

这是一个 **图形编辑器方向的高级前端**，核心工作是构建和优化 Web 端的图像编辑器 + 3D 渲染能力。

从 JD 推测，Anker 大概率在做 **电商素材编辑工具**（产品图编辑、3D 产品展示、营销图文生成等），服务于其跨境电商业务。

### 二、核心能力维度拆解

#### 1. 图形编辑器开发（核心硬性要求）

- **Canvas 2D**：画布操作、图层管理、变换矩阵、事件系统
- **Fabric.js**：基于 Canvas 的编辑器框架，对象模型、序列化、自定义渲染
- **图像处理**：滤镜、裁剪、合成、色彩调整
- **OpenCV.js**：图像识别、边缘检测、抠图等高级处理
- **编辑器架构**：撤销/重做（Command 模式）、图层系统、选择器、对齐/吸附

#### 2. 3D 渲染（重要加分项）

- Three.js / Babylon.js 实际项目
- WebGL 底层理解（shader、渲染管线、纹理）
- 3D 模型加载与展示（glTF/glb）
- 渲染性能优化（帧率、内存、draw call）
- 可能场景：产品 3D 展示、3D 编辑器

#### 3. 复杂前端架构设计（强匹配方向）

- **状态机**：编辑器状态管理（选择/绘制/拖拽/缩放等模式切换）
- **插件化架构**：工具/滤镜/导出格式等可插拔扩展
- **高内聚低耦合**：模块化设计、依赖注入、事件驱动
- 平衡性能、可维护性与用户体验

#### 4. 性能优化（持续要求）

- 编辑器性能：大画布渲染、多图层操作流畅度
- 图像处理性能：大图处理、实时预览
- 3D 渲染性能：帧率、内存
- 通用 Web 性能：首屏、运行时

#### 5. AI 工具使用（基本要求）

- 日常使用 AI 编程工具（Cursor/Claude 等）
- AI 技术工程化应用（加分）
- 门槛不高，但明确写出来说明团队重视

## 补强方案：开源图形编辑器项目

### 项目定位

一个基于 Canvas/Fabric.js 的 Web 图形编辑器，集成 3D 预览和 AI 图像处理能力。

### 为什么这个项目能同时补 JD2 和 JD3 的短板

| 模块 | 覆盖 JD |
|------|---------|
| Fabric.js 画布编辑 | JD3 核心要求 |
| Three.js 3D 预览 | JD2 + JD3 |
| 插件化架构 | JD3 架构要求 |
| 状态机（工具切换） | JD3 架构要求 |
| OpenCV.js 图像处理 | JD3 加分项 |
| 性能优化（大画布/多图层） | JD2 + JD3 |
| TypeScript + React | JD3 技术栈 |

### 核心模块规划

```
graphic-editor/
├── packages/
│   ├── core/          ← 编辑器内核（状态机 + 事件系统 + 插件注册）
│   ├── canvas-2d/     ← Fabric.js 画布层（图层/变换/选择/对齐）
│   ├── renderer-3d/   ← Three.js 3D 预览（模型加载/旋转/光照）
│   ├── image-proc/    ← OpenCV.js 图像处理（滤镜/抠图/调色）
│   ├── plugins/       ← 插件集（文字/形状/导出/AI 生成）
│   └── ui/            ← React UI 层（工具栏/属性面板/图层面板）
├── apps/
│   └── web/           ← 主应用（React + Vite）
└── docs/              ← 架构文档 + 设计决策
```

### 架构亮点（对标 JD 要求）

1. **微内核 + 插件化**：核心只管状态和事件，所有功能（画笔/文字/滤镜/3D）都是插件
2. **状态机驱动**：工具切换、编辑模式、选择状态全部用状态机管理
3. **Command 模式**：撤销/重做系统，操作可序列化
4. **性能优化**：
   - 离屏 Canvas 缓存
   - 脏区域重绘（不全量重绘）
   - 大图分块加载
   - Web Worker 处理图像计算
5. **3D 集成**：Three.js 渲染 3D 模型预览，支持旋转/缩放/光照调整

### 开发优先级

| 阶段 | 内容 | 时间 |
|------|------|------|
| P0 | 核心架构 + Fabric 画布 + 基础编辑（选择/移动/缩放/旋转） | 3-5 天 |
| P1 | 插件系统 + 图层管理 + 撤销重做 | 2-3 天 |
| P2 | Three.js 3D 预览模块 | 2-3 天 |
| P3 | OpenCV.js 图像处理（滤镜/抠图） | 2 天 |
| P4 | 性能优化 + 文档 + README | 1-2 天 |

**总计约 10-15 天**，借助 AI 可以压缩到 7-10 天。

### 项目价值

- 同时覆盖 JD2（Three.js）和 JD3（Canvas/Fabric/编辑器架构）的短板
- 展示架构设计能力在图形领域的落地
- 开源项目 = 可验证的能力证明
- 结合 AI 开发 = 体现 AI 工程化实践
