## 需求拆解

### 一、岗位本质

这是一个 **AIGC/3D 产品方向的高级前端工程师**，核心是在 Vue/Nuxt 技术栈上构建高性能 3D Web 应用。角色定位偏 Tech Lead 方向，需要独立 own 模块 + 带动团队。

从 JD 推测，这家公司大概率是做 **AI 3D 生成**（类似 Meshy、Tripo、Rodin 等方向）的产品，前端需要展示和交互 3D 模型。

### 二、核心能力维度拆解

#### 1. Vue/Nuxt 深度能力（硬性门槛）

- Vue 3 Composition API 熟练使用
- 响应式原理理解（不只是会用，要懂 why）
- 组件设计模式（可复用、可组合）
- Nuxt 4 SSR 实战经验
- 水合机制、跨请求状态隔离
- **关键**：这是 Vue 生态岗，React 经验需要转化表达

#### 2. 3D Web 开发（核心差异化）

- Three.js 实际项目经验
- 3D 模型加载优化（glTF/glb、LOD、懒加载）
- 渲染帧率优化（draw call 合并、frustum culling）
- 内存管理（纹理释放、geometry 复用）
- 与自研引擎集成的能力
- WebGL/WebGPU 理解

#### 3. SSR 与性能（技术深度体现）

- SSR 不是"加一层服务器"，要理解：
  - 水合不匹配问题及解决
  - 跨请求状态隔离（避免状态泄漏）
  - SSR-safe 组件设计
  - 首屏性能优化
- 性能优化要有**数据支撑**：LCP 从 X 降到 Y，内存从 A 降到 B

#### 4. SEO 工程化（业务价值）

- 结构化数据（JSON-LD / Schema.org）
- SSR 友好的 URL 设计
- 动态 meta 管理
- 技术 SEO 对自然流量的影响
- **说明**：这是 to C 产品，SEO 是增长手段

#### 5. 架构与基建能力（Tech Lead 方向）

- 状态管理方案设计（Pinia + 组合式函数）
- API 层抽象
- 自研组件库 / Design System
- 工具链演进
- 制定规范和最佳实践

#### 6. AI 辅助开发（团队文化）

- 日常使用 AI 编程工具
- AI 辅助 Code Review、测试编写
- 能总结可复制的方法论推广到团队
- **注意**：这不是加分项，是基本要求

#### 7. 团队影响力（软实力）

- Code Review 传递经验
- 指导初级成员
- 技术分享和方案评审
- 独立推进复杂项目
