## 学习策略

**方法：源码驱动 + 问题驱动**

不是从头到尾读代码，而是带着问题进入源码：
1. 先理解"这个模块解决什么问题"
2. 再看"它的接口设计是什么"
3. 最后看"核心实现怎么做的"

每个 Phase 产出一篇深度文档（写到 `war2026/` 或 `docs/blog/`），既是学习笔记也是面试素材。

## Phase 3：JS 引擎与 Bridge

**目标：** 理解 JS 和 Native 之间的通信机制，这是跨端框架的核心。

### 要回答的问题
- V8 引擎是怎么集成到 Android 的？
- JS 调用 Native（Feature 调用）的完整链路是什么？
- Native 回调 JS（事件通知）是怎么实现的？
- Bridge 的性能瓶颈在哪？怎么优化？
- 同步调用 vs 异步调用的设计选择？

### 源码入口
```
vendor/platform/android/js-bridge/          ← Bridge 实现
quickapp/core/runtime/.../bridge/           ← Bridge 抽象层
quickapp/external/v8/                       ← V8 引擎
quickapp/core/framework/src/infras/         ← JS 侧 Bridge
```

### 需要产出
- [ ] JS Bridge 通信协议分析（消息格式、序列化方式）
- [ ] V8 集成方案（JNI 调用链、线程模型）
- [ ] 性能优化手段（批量调用、共享内存、序列化优化）

### Android 知识点
- JNI 开发（Native 方法注册、类型映射、引用管理）
- NDK 构建（CMake/ndk-build）
- 线程模型（JS 线程 vs UI 线程 vs IO 线程）
- Handler 机制（跨线程通信）

## Phase 8：跨端方案对比

**目标：** 站在行业高度，对比各跨端方案的设计哲学和 tradeoff。

### 对比维度

| 维度 | 快应用 | 微信小程序 | React Native | Flutter |
|------|--------|-----------|-------------|---------|
| 渲染方式 | Native View | WebView + Native 组件 | Native View | 自绘引擎（Skia） |
| JS 引擎 | V8 | JavaScriptCore | Hermes/JSC | Dart VM |
| 通信机制 | Bridge（同步+异步） | postMessage | Bridge（异步） | Platform Channel |
| 布局引擎 | Yoga (Flexbox) | CSS (WebView) | Yoga | 自研 |
| 热更新 | RPK 包替换 | 包替换 | JS Bundle | 不支持（官方） |
| 系统集成深度 | 系统级预装 | 宿主 App 内 | 独立 App | 独立 App |

### 需要产出
- [ ] 跨端方案技术选型指南
- [ ] 快应用的独特优势与局限性分析

## Android 体系映射

每个 Phase 学完后，你的 Android 知识体系会自然建立：

```
Phase 1 → Application/进程模型/ContentProvider
Phase 2 → Activity 启动/Intent/多进程/TaskAffinity
Phase 3 → JNI/NDK/线程模型/Handler
Phase 4 → View 体系/自定义 View/RecyclerView/硬件加速
Phase 5 → 注解处理/反射/权限/插件化
Phase 6 → 存储/网络/数据库/后台任务
Phase 7 → 性能工具链/R8/内存分析/Baseline Profile
Phase 8 → 行业视野/架构设计能力
```

完成全部 8 个 Phase 后，你具备的能力：
- **跨端框架专家**：能设计和实现一个跨端渲染引擎
- **Android 高级工程师**：系统级开发 + 性能优化 + 架构设计
- **面试竞争力**：有深度的项目经验 + 系统性的知识体系
