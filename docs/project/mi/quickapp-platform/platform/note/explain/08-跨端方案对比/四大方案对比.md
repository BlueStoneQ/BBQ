## 一、核心问题

| 问题 | 本质 |
|------|------|
| 为什么有这么多方案？ | 不同场景的约束不同，没有银弹 |
| 怎么选？ | 看约束条件：性能要求、团队技术栈、发布方式、平台覆盖 |
| 快应用的竞争力在哪？ | 系统级集成 + 免安装 + 原生性能 |

## 二、架构对比

| 维度 | 快应用 | 微信小程序 | React Native | Flutter |
|------|--------|-----------|-------------|---------|
| 渲染 | Native View | WebView + Native 组件 | Native View | Skia 自绘 |
| JS 引擎 | V8 | JavaScriptCore | Hermes | Dart VM |
| 布局 | Yoga (Flexbox) | CSS (WebView) | Yoga | 自研 |
| 宿主 | 系统预装 | 微信 App 内 | 独立 App | 独立 App |
| 分发 | RPK（系统级） | wxapkg（微信内） | App Store | App Store |
| 热更新 | ✅ RPK 替换 | ✅ 包替换 | ✅ JS Bundle | ❌ 官方不支持 |
| 包体 | 0.1-5 MB | 0.1-20 MB | 10-50 MB | 10-50 MB |

## 三、渲染策略对比

### WebView 渲染（小程序）

```
JS 逻辑 → setData() → WebView 渲染
```

- 优点：Web 标准、开发简单
- 缺点：性能天花板低（DOM 操作慢、内存高）
- 小程序的优化：双线程模型（逻辑线程 + 渲染线程分离）

### Native 渲染（快应用/RN）

```
JS 逻辑 → Bridge → Native View
```

- 优点：接近原生性能
- 缺点：Bridge 通信开销、平台差异
- 快应用的优化：J2V8 同步调用、批量 Action

### 自绘渲染（Flutter）

```
Dart 逻辑 → RenderObject → Skia → GPU
```

- 优点：完全一致的跨平台表现、60fps
- 缺点：无法复用平台组件、包体大、不支持热更新

## 四、通信机制对比

| 方案 | JS→Native | Native→JS | 同步支持 | 序列化 |
|------|-----------|-----------|---------|--------|
| 快应用 | V8 JavaCallback | V8 executeFunction | ✅ | JSON / V8Object |
| 小程序 | postMessage | evaluateJS | ❌ | JSON |
| RN (旧) | BatchedBridge | 回调队列 | ❌ | JSON |
| RN (新 JSI) | HostObject | 直接调用 | ✅ | 零拷贝 |
| Flutter | MethodChannel | MethodChannel | ❌ | 二进制编码 |

**快应用和 RN 新架构（JSI）本质相同**：都是把 Native 函数直接注册到 JS 引擎全局环境，实现同步调用。

## 五、适用场景决策树

```
需要免安装？
├── 是 → 需要系统级入口？
│         ├── 是 → 快应用（手机厂商生态）
│         └── 否 → 小程序（超级 App 生态）
└── 否 → 需要极致性能/UI 一致性？
          ├── 是 → Flutter
          └── 否 → 团队是 Web 技术栈？
                    ├── 是 → React Native
                    └── 否 → 原生开发
```

## 六、快应用的独特定位

**快应用 = 系统级小程序**

| 优势 | 原因 |
|------|------|
| 系统级入口 | 预装在 ROM 中，负一屏/应用市场/搜索直达 |
| 免安装 | RPK 动态下载，用完即走 |
| 原生性能 | Native View 渲染，不走 WebView |
| 深度系统集成 | 可以访问 Widget、通知、快捷方式等系统能力 |
| 轻量 | RPK 包通常 < 2MB |

| 局限 | 原因 |
|------|------|
| 生态小 | 只有国内手机厂商联盟支持 |
| 平台绑定 | 只能在支持快应用的手机上运行 |
| 能力受限 | 不如原生 App 能做的事多 |
| 开发者少 | 相比微信小程序/RN/Flutter |
