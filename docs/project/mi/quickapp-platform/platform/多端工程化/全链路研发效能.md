# 全链路研发效能建设

## 能力模型

代码优化 → CI 构建 → 产物管理 → 系统集成 → 设备验证 → 性能分析

## 实践经验

### 1. 代码级优化

- R8/ProGuard 混淆策略设计：多模块 consumerProguardFiles 传递机制分析与优化
- DEX Layout Optimization：基于 baseline profile 的启动内存优化
- shrinkResources + resConfigs：资源精简与多语言裁剪
- 模块化编译时插拔（buildForRom 条件编译）

### 2. CI/CD 流水线

- 多仓库协同构建（3+ 仓库联合编译，change 依赖管理）
- Gerrit 代码审查流程 + cherry-pick + rebase 冲突解决
- 构建参数化（版本号、构建模式、混淆开关等通过 Gradle properties 控制）
- 构建产物签名 + 上传 Maven 仓库 + manifest 版本组合

### 3. 产物管理与系统集成

- APK 产物版本管理（Maven 坐标 + SNAPSHOT 机制）
- ROM manifest 组合（多组件版本拼装成完整系统镜像）
- 动态分区（super.img）打包与 product 分区集成
- 系统级应用预装策略（platform 签名、useLegacyPackaging）

### 4. 设备验证与刷机自动化

- 远程刷机平台操作（fastboot 协议、在线刷机流程）
- adb 设备管理与自动化脚本
- 多设备并行测试（主机 + 辅助机协同）

### 5. 性能分析

- 内存合规测试（老化内存 / 峰值内存 / PSS 分析）
- dumpsys meminfo 深度分析（Java Heap / Native Heap / dex mmap / Code 段拆解）
- APK 体积分析（APK Analyzer、dex/so/assets 逐项拆解）
- 混淆效果验证（seeds.txt 分析、R8 keep 规则调优）

### 6. 工程化方法论

- 优化收益量化：每轮优化前后数据对比，形成可追溯的收益表
- 文档驱动：本地迭代 → 飞书文档 → 评审交付
- 问题排查闭环：现象 → 根因定位 → 修复方案 → 验证 → 记录

## 技术栈

| 领域 | 技术 |
|------|------|
| 构建 | Gradle、R8/ProGuard、AGP 8.x、NDK |
| 版本管理 | Git、Gerrit、rebase/cherry-pick |
| CI/CD | CDP 流水线、Jenkins、Docker 构建环境 |
| 系统 | Android AOSP、动态分区、fastboot、adb |
| 性能 | dumpsys meminfo、PSS/RSS 分析、baseline profile |
| 协作 | 飞书文档、Gerrit Code Review、多团队对齐 |

## 适用方向

- Android 系统级开发 + 工程化
- 嵌入式/IoT 研发效能平台
- 大前端工程化（跨端框架 + 构建优化）
- DevOps / 研发基础设施
