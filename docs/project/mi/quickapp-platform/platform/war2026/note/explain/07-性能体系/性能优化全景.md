## 一、核心问题

| 问题 | 本质 |
|------|------|
| 启动慢怎么办？ | 减少冷启动路径上的工作量 |
| 渲染卡顿怎么办？ | 减少 UI 线程阻塞时间 |
| 内存超标怎么办？ | 减少常驻内存 + 及时释放 |
| 包体太大怎么办？ | 裁剪无用代码/资源 |

## 二、性能指标体系

| 指标 | 含义 | 合规标准 |
|------|------|---------|
| 包体大小 | APK 文件大小 | < 62 MB |
| PSS Max | 进程内存峰值 | < 37.3 MB |
| 冷启动时间 | 点击到首帧渲染 | < 2s |
| 帧率 | 渲染流畅度 | ≥ 55 fps |

## 三、优化手段分层

### 编译时优化

| 手段 | 原理 | 效果 |
|------|------|------|
| R8 混淆 | 删除无用代码 + 缩短类名/方法名 | DEX 42MB → 25MB |
| shrinkResources | 删除无引用的资源文件 | 减少 APK 中的 res/ |
| resConfigs | 只保留指定语言的资源 | 删除 90% 的翻译文件 |
| DEX Layout Optimization | 按启动顺序重排 DEX 中的类 | 减少 page fault |
| ProGuard keep 规则清理 | 移除过宽的 keep 规则 | 让 R8 能裁剪更多代码 |

### 运行时优化

| 手段 | 原理 | 效果 |
|------|------|------|
| V8 Snapshot | 预编译 infras.js 为字节码快照 | 启动时跳过 JS 解析 |
| 进程预热 | 提前 fork Launcher 进程 | 减少冷启动时间 |
| Action 批量发送 | 50 条阈值批量 callNative | 减少 Bridge 调用次数 |
| 异步渲染管线 | JSON 解析在 IO 线程 | UI 线程不阻塞 |
| 组件复用 | RecyclerView 模式 | 长列表内存可控 |

### 构建时优化

| 手段 | 原理 | 效果 |
|------|------|------|
| buildForRom=true | 排除 FinClip 等非必要模块 | APK 减小 80+ MB |
| 16KB 页对齐 | SO 文件对齐到 16KB 边界 | 兼容新内核 |
| NDK 升级 | 使用新版 NDK 编译 SO | 更好的代码生成 |

## 四、你做过的优化（实战经验）

| 优化项 | 方案 | 数据 |
|--------|------|------|
| 包体 152→58 MB | buildForRom + resConfigs + shrinkResources + R8 | -62% |
| DEX 42→25 MB | R8 proguard + consumerProguardFiles 清理 | -41% |
| 内存 55→34 MB | R8 混淆减少 DEX 常驻内存 | -38% |
| 16KB 对齐 | NDK 27 + SO 重编译 + SDK 升级 | 兼容新内核 |

### 踩过的坑

| 坑 | 根因 | 解法 |
|----|------|------|
| R8 不生效 | plugins 模块的 consumerProguardFiles 有宽泛 keep 规则 | 清理 proguard-rules.pro |
| ROM 没带混淆版 APK | manifest 没包含 CI 产物 | 重新上传 CI 产物到 manifest |
| 内存测试超标 | 桌面 Widget 自动拉起快应用到前台 | 移除 Widget 后重测 |
| CDP 构建冲突 | 依赖仓分支配置错误 | 修改 CDP 配置为 ROM 分支 |

## 五、Android 知识映射

| 框架概念 | Android 知识点 |
|----------|---------------|
| R8/ProGuard | 代码混淆、Tree Shaking |
| DEX Layout | Baseline Profile、ART 编译 |
| PSS/RSS | Linux 内存管理、/proc/meminfo |
| 16KB 对齐 | ELF 格式、linker、page size |
| V8 Snapshot | 类比 AOT（dex2oat） |
