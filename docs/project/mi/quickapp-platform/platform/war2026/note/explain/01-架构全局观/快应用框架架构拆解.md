## 一句话理解

快应用框架 = 运行在小米手机系统层的"小程序引擎"。让轻量级应用（RPK 包）无需安装即可运行，类似微信小程序，但直接跑在 Android 系统层面，包名 `com.miui.hybrid`。

## 核心架构

```
┌─────────────────────────────────────────────────────────┐
│                    快应用（RPK 包）                        │
│              开发者用 HTML/CSS/JS 编写                     │
├─────────────────────────────────────────────────────────┤
│  JS 框架层（infras.js）                                   │
│  DSL 解析（Vue/XVM）+ 虚拟 DOM + 样式引擎                  │
├─────────────────────────────────────────────────────────┤
│  JS Bridge（JS ↔ Native 通信）                            │
├─────────────────────────────────────────────────────────┤
│  Android Native 层                                       │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌───────────┐  │
│  │ Runtime  │ │ Features │ │ Widgets  │ │ 小米定制层  │  │
│  │ 渲染引擎  │ │ 系统API  │ │ 原生组件  │ │ overlay   │  │
│  └──────────┘ └──────────┘ └──────────┘ └───────────┘  │
├─────────────────────────────────────────────────────────┤
│  App Shell（com.miui.hybrid）                             │
│  预装在小米手机 /product/app/HybridPlatform/               │
└─────────────────────────────────────────────────────────┘
```

## 三仓库协作

| 仓库 | 路径 | 职责 |
|------|------|------|
| **quickapp** | `packages/quickapp/` | 标准实现：JS 框架、渲染引擎、系统 API、原生组件 |
| **vendor/platform/android** | `packages/vendor/platform/android/` | 主工程（编译入口）：小米定制、最终打包 APK |
| **vendor/plugins/android** | `packages/vendor/plugins/android/` | 小米插件层：账号、支付、推送、广告等 |

编译关系：quickapp 产出 aar → vendor/platform/android 通过 settings.gradle 引用 → 最终打包 APK。

## 快应用启动流程（核心链路）

这是你排查内存问题时走过的链路：

```
外部 Intent（应用市场/负一屏/Widget/deeplink）
    │
    ▼
VendorDispatcherActivity.onCreate()
    │  onPreDispatch() → 预热目标进程
    │  dispatch() → VendorDispatcherHelper.dispatch()
    │
    ▼
VendorDispatcherHelper
    │  getLaunchPkg(intent) → 解析目标快应用包名
    │  performLaunch() → launch()
    │  EntranceController.checkForLaunching() → 云控策略检查（频率限制、黑名单）
    │  EntranceControllerCallback.confirm() → 通过后继续
    │
    ▼
LauncherManager.launch()
    │  选择 Launcher 实例（Launcher0~4，多进程）
    │
    ▼
VendorLauncherActivity（前台 Activity）
    │  显示 Splash → 下载/安装 RPK → 加载 JS → 渲染页面
    │  RootView → V8 执行 infras.js + 应用 JS
    │
    ▼
快应用运行中（JS 通过 Bridge 调用 Native Feature）
```

## 关键组件对照表

| 组件 | 类名 | 作用 | 你接触过的场景 |
|------|------|------|--------------|
| 分发器 | VendorDispatcherActivity | 接收外部 Intent，路由到正确的 Launcher | 内存排查时发现 Widget 触发了它 |
| 启动器 | VendorLauncherActivity$Launcher1 | 前台渲染快应用的 Activity | 内存飙升时它被创建 |
| 宿主服务 | HostService | AIDL 接口，供应用市场等绑定调用 | 应用市场通过它拉起 hybrid |
| 入口控制 | EntranceController | 云控策略：频率限制、黑白名单 | logcat 里看到 ALLOWED/BLOCKED |
| 桌面小部件 | QuickAppWidgetProvider | 桌面 Widget，展示最近使用的快应用 | 内存超标的根因 |
| 分发辅助 | VendorDispatcherHelper | 核心分发逻辑：解析包名、触发启动 | 排查时分析的核心文件 |

## 模块依赖层次

```
app（APK 壳）
 └── app-impl（聚合器，60+ 依赖）
      ├── core-impl（小米核心：分发、启动、HostService、入口控制）
      ├── runtime（quickapp：V8 引擎、页面渲染）
      ├── features（quickapp：系统 API 实现）
      ├── widgets + widgets-miui（原生 UI 组件）
      ├── modules（plugins：插件注册）
      ├── account/pay/push/ad/...（plugins：小米服务）
      ├── game/game-runtime-*（游戏引擎）
      ├── finclip（小程序容器）
      ├── app-widget（桌面小部件）
      └── 第三方适配（wxpay, alipay, share 等）
```

## 构建体系

| 项目 | 值 |
|------|------|
| 构建工具 | Gradle 8.10 + AGP 8.8.2 |
| 语言 | Java/Kotlin 1.9.20 + JavaScript |
| JDK | 17 |
| NDK | 27（16KB 适配后） |
| JS 引擎 | V8（NDK 集成） |
| 构建变体 | phone/tv/soundbox × debug/release |
| 构建命令 | `./gradlew buildHybridPlatform -PbuildForRom=true` |
| 产物 | APK（58MB），预装到 ROM |

## 与你工作的关联

| 你做的事 | 涉及的架构层 |
|----------|------------|
| 包体优化（buildForRom、resConfigs） | app/build.gradle、app-impl 依赖裁剪 |
| R8 proguard 混淆 | 全局 proguard-rules，plugins 的 consumerProguardFiles |
| DEX Layout Optimization | app/build.gradle 的 baselineProfile |
| 内存测试 | 整个进程的 PSS，涉及 Runtime + Features + Native Heap |
| Widget 导致前台拉起 | app-widget 模块 → VendorDispatcherActivity → Launcher |
| 16KB 页对齐 | NDK 升级、SO 重编译、SDK 升级 |
| CDP 构建 | settings.gradle 的三仓库引用关系 |

## Android 知识点映射

通过这个项目你已经接触到的 Android 核心知识：

| 知识点 | 项目中的体现 |
|--------|------------|
| 四大组件 | Activity（Launcher）、Service（HostService）、BroadcastReceiver（WidgetProvider）、ContentProvider（HybridProvider） |
| 进程模型 | 多进程架构（:Launcher0~4, :widgetProvider, :entrance） |
| IPC/AIDL | HostService 暴露 IHostRequest 接口给应用市场 |
| Intent 机制 | deeplink 启动、PendingIntent（Widget 点击） |
| OOM adj | Foreground/Previous/Cached 状态影响内存回收 |
| App Widget | RemoteViews、AppWidgetProvider、定时更新 |
| Gradle 构建 | 多模块、buildVariants、proguard、R8 |
| 性能优化 | R8 混淆、DEX Layout、shrinkResources、16KB 对齐 |
| ROM 集成 | 预装 APK、product 分区、manifest 管理 |
