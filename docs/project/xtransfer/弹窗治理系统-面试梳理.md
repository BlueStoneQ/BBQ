## 一、项目背景与价值

### 1.1 业务痛点
- **弹窗冲突混乱**：多个弹窗同时弹出，用户体验差
- **优先级无序**：重要弹窗（如用户协议）可能被广告弹窗抢占
- **维护困难**：弹窗逻辑分散在各个业务模块，难以统一管理
- **策略调整成本高**：修改弹窗顺序需要改代码、发版本

### 1.2 典型场景

**问题场景**：
```
用户首次登录 App
↓
同时触发：
- 用户协议弹窗（必须同意）
- Android 权限申请弹窗（相机、定位等）
- 广告弹窗（营销活动）
- 引导弹窗（功能介绍）
↓
结果：4 个弹窗同时弹出，用户体验极差
```

### 1.3 解决方案
通过**拦截层 + 调度中心**的分层架构，实现弹窗的统一拦截、优先级排序和顺序展示。

### 1.4 核心价值
- **用户体验提升**：弹窗按优先级顺序展示，避免冲突
- **灵活配置**：支持远程配置弹窗优先级，无需发版
- **易于维护**：统一管理弹窗逻辑，降低维护成本
- **业务解耦**：业务代码无需关心弹窗顺序

## 三、核心技术实现

### 3.1 拦截层设计

#### 3.1.1 HOC 拦截器（Props 驱动弹窗）

**设计思路**：通过高阶组件拦截 `show` prop，将弹窗转换为异步任务

```javascript
export const PopHOC = function(WrappedComponent) {
  return function(props) {
    const { show, ...otherProps } = props;
    const [_show, setShow] = useState(false);
    
    const popTask = {};
    
    // 将弹窗封装为 Promise 任务
    const task = () => {
      return new Promise((resolve, reject) => {
        setShow(true);  // 显示弹窗
        popTask.resolve = resolve;  // 传递给组件，用于关闭时通知
        popTask.reject = reject;
      });
    };
    
    // 拦截 show=true，添加到任务队列
    if (show) {
      taskManager.addTask({
        task,
        type: taskType.引导弹窗
      });
    }
    
    return (
      <WrappedComponent 
        show={_show} 
        popTask={popTask} 
        {...otherProps} 
      />
    );
  };
}
```

**使用方式**：

```javascript
// 原组件
const GuideModal = ({ show, onClose }) => (
  <Modal visible={show} onClose={onClose}>
    引导内容
  </Modal>
);

// 包装后
const ManagedGuideModal = PopHOC(GuideModal);

// 使用
<ManagedGuideModal show={true} />
```

#### 3.1.2 Redux Middleware 拦截器

**设计思路**：拦截触发弹窗的 Action，转换为异步任务

```javascript
const popMiddleware = store => next => action => {
  // 判断是否需要拦截
  if (needInterceptActionTypeList.includes(action.type) && 
      action.payload.from !== 'popManager') {
    
    const task = () => {
      return new Promise((resolve, reject) => {
        // 标记来源，避免重复拦截
        action.payload.from = 'popManager';
        action.payload.$resolve = resolve;
        action.payload.$reject = reject;
        
        // 触发原 Action
        next(action);
      });
    };
    
    taskManager.addTask({
      task,
      type: taskType.广告弹窗
    });
    
    return;  // 拦截，不继续执行
  }
  
  // 不需要拦截的 Action 正常执行
  next(action);
};
```

**Reducer 改造**：

```javascript
function popReducer(state = initialState, action) {
  const { $resolve, $reject } = action.payload;
  
  switch (action.type) {
    case 'SHOW_AD_MODAL':
      return {
        ...state,
        adModal: {
          show: true,
          popTask: { $resolve, $reject }  // 传递给组件
        }
      };
    case 'HIDE_AD_MODAL':
      // 组件关闭时调用 resolve，通知调度中心
      state.adModal.popTask.$resolve();
      return {
        ...state,
        adModal: { show: false }
      };
    default:
      return state;
  }
}
```

#### 3.1.3 AOP 拦截器（原生权限弹窗）

**设计思路**：通过 AOP 重写原生 API，拦截权限申请

```javascript
import { PermissionsAndroid } from 'react-native';

const originalRequest = PermissionsAndroid.request;

PermissionsAndroid.request = (permissionType, config = {}) => {
  // 不在策略中的权限，直接调用
  if (!getStrategy().includes(permissionType)) {
    return originalRequest(permissionType, config);
  }
  
  const task = () => {
    return new Promise((resolve, reject) => {
      originalRequest(permissionType, config)
        .then(res => {
          if (res === PermissionsAndroid.RESULTS.GRANTED) {
            resolve(res);
          } else {
            reject();
          }
        })
        .catch(reject);
    });
  };
  
  taskManager.addTask({
    task,
    type: taskType.permission
  });
};
```

### 3.2 调度中心设计

#### 3.2.1 优先级任务队列（小顶堆）

**数据结构**：

```javascript
class PriorityQueue {
  constructor(compareFn) {
    this.heap = [];
    this.compareFn = compareFn;  // (a, b) => a.priority - b.priority
  }
  
  push(item) {
    this.heap.push(item);
    this._heapifyUp(this.heap.length - 1);
  }
  
  pop() {
    if (this.isEmpty()) return null;
    
    const top = this.heap[0];
    const last = this.heap.pop();
    
    if (this.heap.length > 0) {
      this.heap[0] = last;
      this._heapifyDown(0);
    }
    
    return top;
  }
  
  _heapifyUp(index) {
    while (index > 0) {
      const parentIndex = Math.floor((index - 1) / 2);
      if (this.compareFn(this.heap[index], this.heap[parentIndex]) >= 0) {
        break;
      }
      [this.heap[index], this.heap[parentIndex]] = 
        [this.heap[parentIndex], this.heap[index]];
      index = parentIndex;
    }
  }
  
  _heapifyDown(index) {
    while (true) {
      let minIndex = index;
      const leftChild = 2 * index + 1;
      const rightChild = 2 * index + 2;
      
      if (leftChild < this.heap.length && 
          this.compareFn(this.heap[leftChild], this.heap[minIndex]) < 0) {
        minIndex = leftChild;
      }
      
      if (rightChild < this.heap.length && 
          this.compareFn(this.heap[rightChild], this.heap[minIndex]) < 0) {
        minIndex = rightChild;
      }
      
      if (minIndex === index) break;
      
      [this.heap[index], this.heap[minIndex]] = 
        [this.heap[minIndex], this.heap[index]];
      index = minIndex;
    }
  }
  
  isEmpty() {
    return this.heap.length === 0;
  }
  
  size() {
    return this.heap.length;
  }
}
```

#### 3.2.2 异步任务管理器

**核心逻辑**：

```javascript
const STATUS = {
  RUNNING: 'running',
  IDLE: 'idle'
};

class TaskManager {
  constructor() {
    this.queue = new PriorityQueue((a, b) => a.priority - b.priority);
    this.status = STATUS.IDLE;
    this.strategy = [];
  }
  
  addTask(taskObj) {
    const { type, task } = taskObj;
    
    // 获取优先级
    const priority = getPriority(type);
    
    this.queue.push({ task, priority, type });
    
    // 判断是否应该开始执行
    if (this._shouldStart()) {
      this.run();
    }
  }
  
  run() {
    // 状态机：防止重复执行
    if (this.status === STATUS.RUNNING) return;
    
    // 队列为空，结束
    if (this.queue.isEmpty()) {
      this.status = STATUS.IDLE;
      return;
    }
    
    this.status = STATUS.RUNNING;
    
    const { task } = this.queue.pop();
    
    // 执行任务（Promise）
    task()
      .then(() => {
        this.status = STATUS.IDLE;
        this.run();  // 递归执行下一个任务
      })
      .catch(() => {
        this.status = STATUS.IDLE;
        this.run();  // 失败也继续执行下一个
      });
  }
  
  _shouldStart() {
    // 条件1：所有策略中的弹窗都已加入队列
    const allTasksAdded = this.queue.size() === this.strategy.length;
    
    // 条件2：超时（兜底）
    const timeout = this._isTimeout();
    
    // 条件3：当前空闲
    const isIdle = this.status === STATUS.IDLE;
    
    return isIdle && (allTasksAdded || timeout);
  }
}

export default new TaskManager();
```

#### 3.3.2 策略匹配

```javascript
// 生成策略 Key
export const getStrategyKey = (obj = {
  env: Platform.OS,
  isLogin: isLogin(),
  isFirstLogin: isFirstLogin()
}) => {
  return Object.entries(obj)
    .sort((a, b) => a[0].localeCompare(b[0]))
    .map(([k, v]) => `${k}=${v}`)
    .join('&');
};

// 获取当前策略
export const getStrategy = (strategyKey = getStrategyKey()) => {
  return strategyConfig[strategyKey] || [];
};

// 获取任务优先级
export const getPriority = (taskType, strategy = getStrategy()) => {
  return strategy.indexOf(taskType);
};
```

## 四、技术难点与解决方案

### 4.1 异步任务串行执行

**难点**：如何保证弹窗按顺序一个接一个展示？

**解决方案**：
- 使用 Promise 封装每个弹窗任务
- 任务管理器通过递归 + 状态机控制串行执行
- 当前任务 resolve 后，自动执行下一个任务

### 4.2 不同类型弹窗的统一拦截

**难点**：Props 驱动、Redux 驱动、原生 API 三种弹窗触发方式不同

**解决方案**：
- 针对不同类型设计不同的拦截器
- HOC 拦截 Props
- Middleware 拦截 Redux Action
- AOP 重写原生 API

### 4.3 弹窗关闭通知

**难点**：如何通知调度中心弹窗已关闭，可以执行下一个？

**解决方案**：
- 将 Promise 的 resolve/reject 传递给弹窗组件
- 组件关闭时调用 resolve，通知任务完成
- 调度中心监听 Promise 状态，自动执行下一个任务

### 4.4 策略动态配置

**难点**：如何支持远程配置弹窗优先级，无需发版？

**解决方案**：
- 本地配置作为默认策略
- 启动时从服务端拉取最新策略
- 策略中心支持动态更新

### 4.5 优先级队列性能

**难点**：频繁的插入和删除操作

**解决方案**：
- 使用小顶堆实现优先级队列
- 插入：O(log n)
- 删除：O(log n)
- 弹窗数量有限（一般 < 10），性能足够

## 五、面试高频问题

### 5.1 为什么要做弹窗治理？

**回答要点**：

**回答要点**：

- **调度中心**：优先级队列 + 任务管理器 + 状态机
- **策略中心**：配置化管理弹窗优先级

### 5.3 如何实现不同类型弹窗的拦截？

**回答要点**：
- **Props 驱动**：HOC 拦截 show prop
- **Redux 驱动**：Middleware 拦截 Action

### 5.4 如何保证弹窗按顺序展示？

**回答要点**：
- **优先级队列**：小顶堆，按优先级排序
- **状态机**：RUNNING/IDLE 两种状态，防止并发
- **递归执行**：当前任务完成后，自动执行下一个
- **Promise 链**：通过 Promise 控制异步流程

### 5.5 优先级是如何确定的？

**回答要点**：
- **策略配置**：根据环境、登录状态等匹配策略
- **数组索引**：策略数组的索引即为优先级
- **动态配置**：支持远程配置，无需发版
- **灵活扩展**：可根据业务需求添加新策略

### 5.6 如何通知调度中心弹窗已关闭？

**回答要点**：
- **Promise 传递**：将 resolve/reject 传给组件
- **组件调用**：关闭时调用 resolve
- **自动触发**：调度中心监听 Promise，自动执行下一个
- **错误处理**：reject 也会触发下一个任务

### 5.7 如何支持远程配置？

**回答要点**：
- **本地默认**：本地配置作为默认策略
- **远程拉取**：启动时从服务端获取最新策略
- **动态更新**：策略中心支持运行时更新
- **降级方案**：远程配置失败时使用本地配置

## 七、核心代码示例

### 7.1 完整的任务管理器实现

```javascript
class TaskManager {
  constructor() {
    this.queue = new PriorityQueue((a, b) => a.priority - b.priority);
    this.status = 'idle';
  }
  
  addTask({ type, task }) {
    const priority = getPriority(type);
    this.queue.push({ task, priority });
    
    if (this.status === 'idle') {
      this.run();
    }
  }
  
  run() {
    if (this.queue.isEmpty()) {
      this.status = 'idle';
      return;
    }
    
    this.status = 'running';
    const { task } = this.queue.pop();
    
    task()
      .then(() => {
        this.status = 'idle';
        this.run();
      })
      .catch(() => {
        this.status = 'idle';
        this.run();
      });
  }
}
```
