## 一、性能指标体系 ⭐⭐⭐ 🔥🔥🔥

### 1.1 核心性能指标 ⭐⭐⭐ 🔥🔥🔥

**考点**：前端性能的关键指标

**渲染流程与指标**：
```
Network（网络请求）
    ↓ 首字节时间（TTFB）
Parsing（解析 HTML）
    ↓ DOM 构建完成
Style（样式计算）
    ↓ DOM Ready
Paint（绘制）
    ↓ FP / FCP
Composite（合成）
    ↓ 页面完全加载（Load）
```

**核心指标详解**：

#### 1. FP（First Paint）⭐⭐⭐ 🔥
- **定义**：白屏时间，从页面开始加载到浏览器首次渲染
- **计算**：`FP = firstPaint - pageStartTime`
- **意义**：用户看到页面有反应的时间

#### 2. FCP（First Contentful Paint）⭐⭐⭐ 🔥🔥
- **定义**：首次内容绘制，页面内容首次呈现在屏幕上
- **标准**：1.8s 以内为优秀
- **计算**：`FCP = firstContentTime - pageStartTime`
- **区别**：FP 是任何渲染，FCP 是有意义的内容（文本、图像）

#### 3. FMP（First Meaningful Paint）⭐⭐⭐ 🔥🔥
- **定义**：首次有效绘制，页面主要内容开始出现
- **标准**：< 100ms
- **特点**：因页面而异，没有统一规范
- **上报**：一般在页面主接口的 setData 回调中手动上报

#### 4. LCP（Largest Contentful Paint）⭐⭐⭐ 🔥🔥
- **定义**：最大内容绘制，视窗最大可见元素的渲染时间
- **标准**：2.5s 以内为优秀
- **测量**：使用 `web-vitals` 库
```javascript
import { getLCP } from 'web-vitals';

getLCP(console.log);
```

#### 5. TTI（Time To Interactive）⭐⭐⭐ 🔥🔥
- **定义**：首次可交互时间，页面可以快速响应用户输入
- **计算**：`TTI = domInteractive - fetchStart`
- **测量**：使用 `tti-polyfill`

#### 6. FST（First Screen Time）⭐⭐⭐ 🔥
- **定义**：秒开率，首屏加载时间
- **计算**：`首屏时间 = 页面完全加载时间 - 开始响应 URL 时间`

### 1.2 Performance API ⭐⭐⭐ 🔥🔥

**考点**：如何获取性能数据

**Navigation Timing API**：
```javascript
// 获取性能时间点
const timing = window.performance.timing;

// 常用指标计算
const metrics = {
  // 首字节时间（TTFB）
  ttfb: timing.responseStart - timing.domainLookupStart,
  
  // 白屏时间（FP）
  fp: timing.responseEnd - timing.fetchStart,
  
  // 首次可交互时间（TTI）
  tti: timing.domInteractive - timing.fetchStart,
  
  // DOM Ready 时间
  domReady: timing.domContentLoadedEventEnd - timing.fetchStart,
  
  // 页面完全加载时间（Load）
  load: timing.loadEventStart - timing.fetchStart
};
```

**Performance Observer**：
```javascript
// 监听 FCP
const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    if (entry.name === 'first-contentful-paint') {
      console.log('FCP:', entry.startTime);
    }
  }
});

observer.observe({ entryTypes: ['paint'] });
```

### 1.3 SPA 下的首屏时间 ⭐⭐⭐ 🔥🔥

**考点**：单页应用的性能监控

**问题**：
- W3C 标准无法完全满足 SPA 的监控需求
- 页面内容渲染时机被改变

**解决方案**：
1. **手动上报 FMP**：在主接口回调中上报
2. **MutationObserver**：监听 DOM 变化
3. **自定义打点**：关键节点手动打点

```javascript
// 手动上报 FMP
fetchMainData().then(data => {
  this.setData(data, () => {
    // 在 setData 回调中上报 FMP
    reportFMP(Date.now() - pageStartTime);
  });
});
```

## 二、网络层优化 ⭐⭐⭐ 🔥🔥🔥

### 2.1 资源加载优化 ⭐⭐⭐ 🔥🔥

**考点**：如何优化资源加载

**优化策略**：
```
DNS 解析
    ↓
TCP 连接
    ↓
HTTP 请求
    ↓ 减少请求次数、减少请求体积
服务端处理
    ↓
客户端接收
    ↓ 解析渲染
```

#### 2.1.1 懒加载（Lazy Load）⭐⭐⭐ 🔥🔥

**考点**：图片懒加载的实现原理

**核心思想**：
- 只加载视口内的资源
- 滚动时动态加载进入视口的资源

**实现原理**：
```javascript
// 1. 获取视口高度
const viewHeight = window.innerHeight || document.documentElement.clientHeight;

// 2. 获取元素距离视口顶部的距离
const top = element.getBoundingClientRect().top;

// 3. 判断是否进入视口
if (top <= viewHeight) {
  // 加载图片
  img.src = img.getAttribute('data-src');
}
```

**完整实现**：
```javascript
const imgs = document.querySelectorAll('img[data-src]');
const viewHeight = window.innerHeight || document.documentElement.clientHeight;
let imgStartIndex = 0;

const imgLazyLoad = function() {
  // 遍历未加载的图片
  for (let i = imgStartIndex; i < imgs.length; i++) {
    const curImg = imgs[i];
    // 计算距离视口底部的距离
    const distance = curImg.getBoundingClientRect().top - viewHeight;
    
    // 进入视口（可以设置阈值提前加载）
    if (distance <= 0) {
      curImg.src = curImg.getAttribute('data-src');
      imgStartIndex++;
    }
  }
};

// 注意：必须使用节流处理
window.addEventListener('scroll', throttle(imgLazyLoad, 200));
```

**现代方案：Intersection Observer**：
```javascript
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.getAttribute('data-src');
      observer.unobserve(img);
    }
  });
});

imgs.forEach(img => observer.observe(img));
```

#### 2.1.2 预加载（Preload）⭐⭐⭐ 🔥

**考点**：资源预加载的方式

**1. Link 标签预加载**：
```html
<!-- 预加载关键资源 -->
<link rel="preload" href="style.css" as="style">
<link rel="preload" href="main.js" as="script">
<link rel="preload" href="font.woff2" as="font" crossorigin>

<!-- 预获取未来可能需要的资源 -->
<link rel="prefetch" href="next-page.js">
```

**区别**：
- **preload**：当前页面必需的资源，高优先级
- **prefetch**：未来可能需要的资源，低优先级

**2. DNS 预解析**：
```html
<link rel="dns-prefetch" href="//api.example.com">
```

**3. 预连接**：
```html
<link rel="preconnect" href="//cdn.example.com">
```

#### 2.1.3 接口聚合 ⭐⭐ 🔥

**考点**：减少请求次数

**问题**：
- 多个接口请求，增加网络开销
- 串行请求，影响加载速度

**解决方案**：
```javascript
// 后端提供聚合接口
GET /api/aggregate?apis=user,config,menu

// 前端一次请求获取多个数据
const { user, config, menu } = await fetchAggregate();
```

### 2.2 HTTP 缓存策略 ⭐⭐⭐ 🔥🔥🔥

**考点**：HTTP 缓存的工作原理

**缓存流程**：
```
浏览器请求资源
    ↓
检查强缓存
    ├─ 命中 → 直接使用缓存
    └─ 未命中 ↓
检查协商缓存
    ├─ 304 → 使用缓存
    └─ 200 → 下载新资源
```

#### 2.2.1 强缓存 ⭐⭐⭐ 🔥🔥

**特点**：不需要向服务器确认，直接使用缓存

**1. Expires（HTTP/1.0）**：
```
Expires: Wed, 21 Oct 2025 07:28:00 GMT
```
- 绝对时间点
- 依赖本地时间，可能不准确

**2. Cache-Control（HTTP/1.1）**：
```
Cache-Control: max-age=31536000
```
- 相对时间段（秒）
- 优先级高于 Expires
- 更准确，推荐使用

**常用指令**：
- `max-age=<seconds>`：缓存时间
- `no-cache`：需要协商缓存
- `no-store`：不缓存
- `public`：可被任何缓存
- `private`：只能被浏览器缓存

#### 2.2.2 协商缓存 ⭐⭐⭐ 🔥🔥

**特点**：需要向服务器确认资源是否更新

**1. Last-Modified / If-Modified-Since（基于时间）**：
```
# 首次请求
Response Headers:
Last-Modified: Wed, 21 Oct 2024 07:28:00 GMT

# 再次请求
Request Headers:
If-Modified-Since: Wed, 21 Oct 2024 07:28:00 GMT

# 服务器响应
- 未修改：304 Not Modified
- 已修改：200 OK + 新资源 + 新 Last-Modified
```

**缺点**：
- 只能精确到秒
- 文件修改但内容未变，也会重新下载

**2. ETag / If-None-Match（基于内容）**：
```
# 首次请求
Response Headers:
ETag: "33a64df551425fcc55e4d42a148795d9f25f89d4"

# 再次请求
Request Headers:
If-None-Match: "33a64df551425fcc55e4d42a148795d9f25f89d4"

# 服务器响应
- 匹配：304 Not Modified
- 不匹配：200 OK + 新资源 + 新 ETag
```

**优点**：
- 基于内容哈希，更准确
- 内容不变，即使修改时间变了也不会重新下载

**优先级**：
- ETag 优先级高于 Last-Modified
- 两者同时存在时，以 ETag 为准

## 三、渲染层优化 ⭐⭐⭐ 🔥🔥🔥

### 3.1 虚拟列表 ⭐⭐⭐ 🔥🔥🔥

**考点**：长列表性能优化方案

**核心思想**：
- 只渲染可见区域的元素
- 滚动时动态更新渲染内容

**实现原理**：
```
┌─────────────────────┐
│   缓冲区（上）       │ ← 提前渲染
├─────────────────────┤
│                     │
│   可见区域          │ ← 当前视口
│                     │
├─────────────────────┤
│   缓冲区（下）       │ ← 提前渲染
└─────────────────────┘
```

**关键计算**：
```javascript
// 1. 可见区域高度
const viewHeight = window.innerHeight;

// 2. 每项高度（固定高度）
const itemHeight = 50;

// 3. 可见数量
const visibleCount = Math.ceil(viewHeight / itemHeight);

// 4. 开始索引
const startIndex = Math.floor(scrollTop / itemHeight);

// 5. 结束索引
const endIndex = startIndex + visibleCount;

// 6. 渲染数据
const visibleData = allData.slice(startIndex, endIndex);

// 7. 偏移量（保持滚动位置）
const offsetY = startIndex * itemHeight;
```

**完整实现**：
```javascript
class VirtualList {
  constructor(options) {
    this.container = options.container;
    this.data = options.data;
    this.itemHeight = options.itemHeight;
    this.render = options.render;
    
    this.viewHeight = this.container.clientHeight;
    this.visibleCount = Math.ceil(this.viewHeight / this.itemHeight);
    
    this.init();
  }
  
  init() {
    // 创建容器
    this.phantom = document.createElement('div');
    this.phantom.style.height = `${this.data.length * this.itemHeight}px`;
    
    this.content = document.createElement('div');
    this.content.style.transform = 'translate3d(0, 0, 0)';
    
    this.container.appendChild(this.phantom);
    this.container.appendChild(this.content);
    
    // 监听滚动
    this.container.addEventListener('scroll', () => {
      this.update();
    });
    
    this.update();
  }
  
  update() {
    const scrollTop = this.container.scrollTop;
    const startIndex = Math.floor(scrollTop / this.itemHeight);
    const endIndex = startIndex + this.visibleCount;
    
    // 渲染可见数据
    const visibleData = this.data.slice(startIndex, endIndex);
    this.content.innerHTML = visibleData.map(this.render).join('');
    
    // 设置偏移
    const offsetY = startIndex * this.itemHeight;
    this.content.style.transform = `translate3d(0, ${offsetY}px, 0)`;
  }
}
```

**使用库**：
- React：`react-window`、`react-virtualized`
- Vue：`vue-virtual-scroller`

### 3.2 CSS 性能优化 ⭐⭐⭐ 🔥🔥

**考点**：CSS 对性能的影响

#### 3.2.1 选择器优化 ⭐⭐⭐ 🔥

**CSS 匹配顺序**：从右到左

```css
/* ❌ 性能差：需要遍历所有元素 */
* {
  margin: 0;
}

/* ❌ 性能差：需要遍历所有 div */
div * {
  color: red;
}

/* ✅ 性能好：直接匹配类 */
.container {
  color: red;
}
```

**优化建议**：
1. 避免使用通配符 `*`
2. 减少嵌套层级
3. 优先使用类选择器
4. 避免使用标签选择器（如果可以用类代替）

#### 3.2.2 GPU 加速 ⭐⭐⭐ 🔥🔥

**触发 GPU 加速的属性**：
```css
/* 使用 transform */
.element {
  transform: translateZ(0);
  /* 或 */
  transform: translate3d(0, 0, 0);
}

/* 使用 will-change */
.element {
  will-change: transform, opacity;
}
```

**注意事项**：
- 不要滥用，会占用大量内存
- 动画结束后移除 `will-change`

```javascript
el.addEventListener('mouseenter', () => {
  el.style.willChange = 'transform, opacity';
});

el.addEventListener('animationend', () => {
  el.style.willChange = 'auto';
});
```

### 3.3 回流与重绘 ⭐⭐⭐ 🔥🔥🔥

**考点**：如何减少回流和重绘

**回流（Reflow）**：
- 改变几何属性（位置、尺寸）
- 改变 DOM 树结构
- 获取某些属性（offsetTop、scrollTop 等）

**重绘（Repaint）**：
- 改变颜色等非几何属性

**性能影响**：
```
回流 > 重绘 > 合成
```

**优化策略**：

#### 1. 批量修改样式 ⭐⭐⭐ 🔥
```javascript
// ❌ 逐条修改，触发多次回流
el.style.width = '100px';
el.style.height = '100px';
el.style.margin = '10px';

// ✅ 使用 class 批量修改
el.className = 'new-style';

// ✅ 使用 cssText
el.style.cssText = 'width: 100px; height: 100px; margin: 10px;';
```

#### 2. DOM 离线操作 ⭐⭐⭐ 🔥
```javascript
// ❌ 直接操作 DOM
for (let i = 0; i < 1000; i++) {
  const li = document.createElement('li');
  li.textContent = i;
  ul.appendChild(li);  // 触发 1000 次回流
}

// ✅ 使用 DocumentFragment
const fragment = document.createDocumentFragment();
for (let i = 0; i < 1000; i++) {
  const li = document.createElement('li');
  li.textContent = i;
  fragment.appendChild(li);
}
ul.appendChild(fragment);  // 只触发 1 次回流

// ✅ 使用 innerHTML
const html = Array.from({ length: 1000 }, (_, i) => `<li>${i}</li>`).join('');
ul.innerHTML = html;
```

#### 3. 避免频繁读取布局信息 ⭐⭐⭐ 🔥
```javascript
// ❌ 读写交替，触发多次回流
el.style.width = el.offsetWidth + 10 + 'px';
el.style.height = el.offsetHeight + 10 + 'px';

// ✅ 先读后写
const width = el.offsetWidth;
const height = el.offsetHeight;
el.style.width = width + 10 + 'px';
el.style.height = height + 10 + 'px';
```

#### 4. 使用 transform 代替 top/left ⭐⭐⭐ 🔥🔥
```css
/* ❌ 触发回流 */
.element {
  position: absolute;
  left: 100px;
  top: 100px;
}

/* ✅ 只触发合成 */
.element {
  transform: translate(100px, 100px);
}
```

### 3.4 动画性能优化 ⭐⭐⭐ 🔥🔥

**考点**：如何实现高性能动画

#### 3.4.1 CSS 动画优化 ⭐⭐⭐ 🔥

**优化原则**：
1. 使用 `transform` 和 `opacity`（只触发合成）
2. 避免使用 `width`、`height`、`left`、`top`（触发回流）
3. 使用 `will-change` 提前告知浏览器

```css
/* ✅ 高性能动画 */
.element {
  will-change: transform, opacity;
  animation: slide 1s ease-in-out;
}

@keyframes slide {
  from {
    transform: translateX(0);
    opacity: 0;
  }
  to {
    transform: translateX(100px);
    opacity: 1;
  }
}
```

**独立图层**：
```css
/* 将动画元素提升到独立图层 */
.element {
  position: absolute;  /* 或 fixed */
  transform: translateZ(0);  /* 或 translate3d(0, 0, 0) */
}
```

#### 3.4.2 JS 动画优化 ⭐⭐⭐ 🔥

**使用 requestAnimationFrame**：
```javascript
// ❌ 使用 setInterval
setInterval(() => {
  element.style.left = element.offsetLeft + 1 + 'px';
}, 16);

// ✅ 使用 requestAnimationFrame
function animate() {
  element.style.left = element.offsetLeft + 1 + 'px';
  requestAnimationFrame(animate);
}
requestAnimationFrame(animate);
```

**优势**：
- 与浏览器刷新率同步（60fps）
- 页面不可见时自动暂停
- 性能更好

### 3.5 SSR 服务端渲染 ⭐⭐⭐ 🔥🔥

**考点**：SSR 的优势和实现

**CSR vs SSR**：
```
CSR（客户端渲染）：
服务器 → HTML 壳 → 浏览器 → 下载 JS → 执行 JS → 渲染页面

SSR（服务端渲染）：
服务器 → 完整 HTML → 浏览器 → 直接显示 → 下载 JS → 激活
```

**优势**：
1. **SEO 友好**：搜索引擎可以抓取完整内容
2. **首屏速度快**：直接返回渲染好的 HTML
3. **用户体验好**：更快看到内容

**缺点**：
- 服务器压力大
- 开发复杂度高
- 不适合高交互应用

**Vue SSR 实现**：

### 3.6 节流与防抖 ⭐⭐⭐ 🔥🔥

**考点**：高频事件的性能优化

#### 3.6.1 防抖（Debounce）⭐⭐⭐ 🔥

**定义**：事件触发后延迟执行，如果在延迟期间再次触发，则重新计时

**应用场景**：
- 搜索框输入
- 窗口 resize
- 表单验证

**实现**：
```javascript
function debounce(fn, delay) {
  let timer = null;
  
  return function(...args) {
    // 清除之前的定时器
    if (timer) clearTimeout(timer);
    
    // 设置新的定时器
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}

// 使用
const handleInput = debounce((e) => {
  console.log('搜索:', e.target.value);
}, 500);

input.addEventListener('input', handleInput);
```

#### 3.6.2 节流（Throttle）⭐⭐⭐ 🔥

**定义**：固定时间内只执行一次

**应用场景**：
- 滚动事件
- 鼠标移动
- 按钮点击

**实现**：
```javascript
function throttle(fn, delay) {
  let lastTime = 0;
  
  return function(...args) {
    const now = Date.now();
    
    if (now - lastTime >= delay) {
      fn.apply(this, args);
      lastTime = now;
    }
  };
}

// 使用
const handleScroll = throttle(() => {
  console.log('滚动位置:', window.scrollY);
}, 200);

window.addEventListener('scroll', handleScroll);
```

#### 3.6.3 有底线的防抖 ⭐⭐ 🔥🔥

**需求**：结合防抖和节流，既要延迟执行，又要保证最长等待时间

```javascript
function debounceWithMaxWait(fn, delay, maxWait) {
  let timer = null;
  let lastTime = 0;
  
  return function(...args) {
    const now = Date.now();
    
    // 清除之前的定时器
    if (timer) clearTimeout(timer);
    
    // 超过最长等待时间，立即执行
    if (now - lastTime >= maxWait) {
      fn.apply(this, args);
      lastTime = now;
    } else {
      // 否则延迟执行
      timer = setTimeout(() => {
        fn.apply(this, args);
        lastTime = Date.now();
      }, delay);
    }
  };
}
```

```javascript
class ErrorMonitor {
  constructor() {
    this.init();
  }
  
  init() {
    // 1. 捕获全局错误
    window.onerror = (message, source, lineno, colno, error) => {
      this.report({
        type: 'error',
        message,
        source,
        lineno,
        colno,
        stack: error?.stack
      });
    };
    
    // 2. 捕获 Promise 错误
    window.addEventListener('unhandledrejection', (event) => {
      this.report({
        type: 'promise',
        reason: event.reason
      });
    });
    
    // 3. 捕获资源加载错误
    window.addEventListener('error', (event) => {
      if (event.target !== window) {
        this.report({
          type: 'resource',
          tagName: event.target.tagName,
          src: event.target.src || event.target.href
        });
      }
    }, true);
  }
  
  report(data) {
    // 上报错误
    navigator.sendBeacon('/api/error', JSON.stringify(data));
  }
}
```

## 七、高频面试题 ⭐⭐⭐

### Q1: 说说前端性能优化的手段？🔥🔥🔥

**答**：
性能优化可以从以下几个方面入手：

**1. 网络层优化**：
- 减少请求次数：合并资源、雪碧图、接口聚合
- 减少请求体积：Gzip 压缩、图片压缩、代码压缩
- 使用缓存：HTTP 缓存、CDN 缓存
- 懒加载：图片懒加载、路由懒加载
- 预加载：preload、prefetch、DNS 预解析

**2. 渲染层优化**：
- 减少回流重绘：批量修改样式、使用 transform
- 虚拟列表：长列表只渲染可见部分
- 防抖节流：优化高频事件
- CSS 优化：减少选择器层级、使用 GPU 加速
- SSR：服务端渲染，提升首屏速度

**3. 构建优化**：
- 代码分割：SplitChunks、动态导入
- Tree Shaking：删除未使用代码
- 压缩代码：JS/CSS/HTML 压缩
- 使用缓存：Webpack 缓存、Babel 缓存

**4. 用户体验**：
- 骨架屏：减少白屏感知
- Loading 动画：给用户反馈
- 渐进式加载：先显示主要内容

### Q2: 什么是 FCP、LCP、TTI？🔥🔥🔥

**答**：
- **FCP（First Contentful Paint）**：首次内容绘制
  - 页面内容首次呈现在屏幕上的时间
  - 标准：1.8s 以内

- **LCP（Largest Contentful Paint）**：最大内容绘制
  - 视窗最大可见元素的渲染时间
  - 标准：2.5s 以内

- **TTI（Time To Interactive）**：首次可交互时间
  - 页面可以快速响应用户输入的时间
  - 没有耗时长任务

这些是 Web Vitals 的核心指标，用于衡量用户体验。

### Q3: 如何实现图片懒加载？🔥🔥🔥

**答**：
**原理**：
1. 图片 src 先设置为占位图
2. 真实地址存在 data-src 中
3. 监听滚动事件
4. 判断图片是否进入视口
5. 进入视口后，将 data-src 赋值给 src

**实现**：
```javascript
const imgs = document.querySelectorAll('img[data-src]');
const viewHeight = window.innerHeight;

function lazyLoad() {
  imgs.forEach(img => {
    const rect = img.getBoundingClientRect();
    if (rect.top <= viewHeight) {
      img.src = img.dataset.src;
    }
  });
}

window.addEventListener('scroll', throttle(lazyLoad, 200));
```

**现代方案**：使用 Intersection Observer API。

### Q4: 什么是回流和重绘？如何优化？🔥🔥🔥

**答**：
**回流（Reflow）**：
- 元素的几何属性（位置、尺寸）改变
- DOM 树结构改变
- 触发条件：修改 width、height、margin、padding 等

**重绘（Repaint）**：
- 元素的外观改变，但几何属性不变
- 触发条件：修改 color、background、visibility 等

**性能影响**：回流 > 重绘 > 合成

**优化方法**：
1. 批量修改样式（使用 class）
2. DOM 离线操作（DocumentFragment）
3. 使用 transform 代替 top/left
4. 避免频繁读取布局信息
5. 使用 will-change 提前告知浏览器

### Q5: 如何实现虚拟列表？🔥🔥🔥

**答**：
**核心思想**：只渲染可见区域的元素

**关键计算**：
1. 可见区域高度：`window.innerHeight`
2. 每项高度：固定或动态
3. 可见数量：`Math.ceil(viewHeight / itemHeight)`
4. 开始索引：`Math.floor(scrollTop / itemHeight)`
5. 渲染数据：`data.slice(startIndex, endIndex)`
6. 偏移量：`startIndex * itemHeight`

**实现要点**：
- 创建占位容器（撑开滚动高度）
- 创建内容容器（渲染可见元素）
- 监听滚动事件（更新渲染内容）
- 使用 transform 设置偏移

### Q6: HTTP 缓存策略有哪些？🔥🔥🔥

**答**：
**强缓存**（不需要向服务器确认）：
- **Expires**：绝对时间点（HTTP/1.0）
- **Cache-Control**：相对时间段（HTTP/1.1，优先级更高）

**协商缓存**（需要向服务器确认）：
- **Last-Modified / If-Modified-Since**：基于时间
- **ETag / If-None-Match**：基于内容（优先级更高）

**缓存流程**：
1. 检查强缓存，命中则直接使用
2. 未命中，检查协商缓存
3. 服务器返回 304（使用缓存）或 200（下载新资源）

### Q7: 什么是防抖和节流？🔥🔥

**答**：
**防抖（Debounce）**：
- 事件触发后延迟执行
- 延迟期间再次触发，重新计时
- 应用：搜索框输入、窗口 resize

**节流（Throttle）**：
- 固定时间内只执行一次
- 应用：滚动事件、鼠标移动

**区别**：
- 防抖：只执行最后一次
- 节流：固定频率执行

### Q8: 如何优化首屏加载速度？🔥🔥🔥

**答**：
**1. 资源优化**：
- 代码分割，减小首屏 JS 体积
- 图片压缩、使用 WebP
- 开启 Gzip 压缩
- 使用 CDN

**2. 渲染优化**：
- SSR 服务端渲染
- 骨架屏
- 关键 CSS 内联
- 预加载关键资源

**3. 网络优化**：
- HTTP/2
- DNS 预解析
- 使用缓存

**4. 代码优化**：
- Tree Shaking
- 按需加载
- 懒加载

### Q9: 什么是 Tree Shaking？🔥🔥

**答**：
Tree Shaking 是删除未使用代码的优化技术。

**原理**：
- 基于 ES6 模块的静态分析
- 编译时确定模块依赖关系
- 删除未引用的导出

**使用条件**：
1. 使用 ES6 模块（import/export）
2. 生产模式（mode: 'production'）
3. 配置 sideEffects

**注意**：CommonJS 模块无法 tree-shake。

### Q10: 如何监控前端性能？🔥🔥

**答**：
**1. 性能指标采集**：
- Performance API：获取性能时间点
- Performance Observer：监听性能事件
- 自定义打点：关键节点手动上报

**2. 异常捕获**：
- try-catch：捕获同步错误
- window.onerror：捕获全局错误
- unhandledrejection：捕获 Promise 错误

**3. 数据上报**：
- Beacon API：页面卸载时上报
- Image 请求：兼容性好
- XHR/Fetch：功能强大

**4. 数据分析**：
- 性能大盘：展示核心指标
- 告警系统：异常及时通知
- 报表分析：趋势分析、对比分析

### Q11: SSR 的优缺点是什么？🔥🔥

**答**：
**优点**：
1. SEO 友好：搜索引擎可以抓取完整内容
2. 首屏速度快：直接返回渲染好的 HTML
3. 用户体验好：更快看到内容

**缺点**：
1. 服务器压力大：每次请求都要渲染
2. 开发复杂度高：需要考虑服务端和客户端
3. 不适合高交互应用：频繁交互会增加服务器负担

**适用场景**：
- 内容型网站（博客、新闻）
- 需要 SEO 的页面
- 首屏性能要求高的应用

### Q12: 如何优化动画性能？🔥🔥

**答**：
**CSS 动画优化**：
1. 使用 transform 和 opacity（只触发合成）
2. 避免使用 width、height、left、top（触发回流）
3. 使用 will-change 提前告知浏览器
4. 将动画元素提升到独立图层

**JS 动画优化**：
1. 使用 requestAnimationFrame
2. 避免在动画中读取布局信息
3. 使用 Web Worker 处理复杂计算

**通用优化**：
- 减少动画元素数量
- 降低动画复杂度
- 使用 CSS 动画代替 JS 动画

### Q13: 什么是骨架屏？如何实现？🔥🔥

**答**：
**作用**：
- 减少白屏时间的感知
- 提升用户体验
- 给用户加载中的反馈

**实现方式**：
1. **手写 HTML/CSS**：简单直接
2. **使用图片**：快速实现
3. **自动生成**：使用 Webpack 插件

**最佳实践**：
- 骨架屏应该与真实内容结构相似
- 使用动画效果（如渐变）
- 加载完成后平滑过渡

### Q14: 如何减少白屏时间？🔥🔥

**答**：
**原因**：
- JS 文件过大，下载和执行时间长
- 首屏渲染依赖 JS 执行
- 网络慢，资源加载慢

**解决方案**：
1. **资源优化**：代码分割、CDN、Gzip
2. **渲染优化**：SSR、骨架屏、Loading
3. **预加载**：preload 关键资源
4. **内联关键 CSS**：首屏 CSS 内联到 HTML

### Q15: 如何优化长列表性能？🔥🔥

**答**：
**问题**：
- 渲染大量 DOM 元素
- 占用大量内存
- 滚动卡顿

**解决方案**：
1. **虚拟列表**：只渲染可见部分
2. **分页加载**：滚动到底部加载更多
3. **懒加载**：图片等资源懒加载
4. **防抖节流**：优化滚动事件

**推荐库**：
- React：react-window、react-virtualized
- Vue：vue-virtual-scroller
