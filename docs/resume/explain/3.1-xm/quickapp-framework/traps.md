## 雷 1：你在框架里的角色

**可能的问法**："框架是你设计的还是你接手的？你具体负责什么？"

**回答**：

框架是团队多年积累的产品，我加入后负责**性能治理和工程化演进**：
- 包体 153MB → 60MB（模块裁剪方案设计）
- 启动内存 PSS 41MB → 35.8MB（DEX 布局优化）
- 自动化测试体系搭建（三端设备自动化）
- 条件编译工具套件（开源）

做这些事的前提是**深入理解框架整体架构**（三线程模型/J2V8 Bridge/渲染引擎），才能知道瓶颈在哪、该从哪一层优化。

**关键表达**：
- 不说"我从零设计的"（诚实）
- 强调"做优化必须深入理解架构"（证明有框架级认知）
- 把"优化"讲成"治理/演进"（架构师视角，不是执行者）
