## 背景

系统级快应用框架（预装 ROM），OS4 合规审查老化内存峰值（PSS Max）超标。合规要求 < 37.3MB，实测 41.9MB，超标 12.3%。

## 核心思路：减少 dex 常驻内存

dex 体积大（42MB+）→ 代码段 mmap 常驻内存 → PSS 高。缩小 dex = 降低内存。

## 三管齐下

| 优化项 | 原理 | 收益 |
|--------|------|------|
| R8 严格混淆 | 全局混淆删除未使用代码，缩短类名/方法名 | dex -39% |
| DEX Layout Optimization | 基于 baseline profile 重排 dex，热代码集中前几页，冷代码延迟加载 | 启动内存降低 |
| shrinkResources | 移除未引用资源 | 资源段缩小 |

## 结果数据

**dex 体积：**
- 优化前：42.33 MB（6 个 dex）
- 优化后：27.05 MB（4 个 dex）
- 收益：-15.28 MB（-36%）

**内存占用（dumpsys meminfo）：**
- .dex mmap：25.6 MB → 17.8 MB（-7.8 MB）
- TOTAL PSS：43.1 MB → 33.4 MB（-9.7 MB，-22.5%）

## 关键技术决策

- minifyEnabled 只在 app 模块开启（全局 R8 分析效果优于各模块独立混淆）
- 清理 consumerProguardFiles 中的宽泛规则，但保留 consumer 机制
- 涉及 3 个仓库、9 笔提交

## 验证方式

- 本地构建验证 dex 体积
- CI 构建 → ROM 集成 → 刷机 → dumpsys meminfo 对比
- 老化内存测试环境：48 个应用预置 + 自动化测试工具 + 1000+ 用例 / 5-6 小时/轮
