## 二、开源项目

### 条件编译工具套件

面向前端主流源码类型（xml, css, js），基于 AOT 以特殊注释作为编译指令进行条件编译，实现对源码内容根据不同目标平台进行精确裁剪，减少打包体积，减轻运行时压力。可用于内存/性能优化、多端复用/跨端/跨平台方案。

- 面向 js：babel-plugin-conditional-compile-with-comment
- 面向 css：postcss-plugin-conditional-compile
- 面向 xml 模版：xml-conditional-compile

- 背景：自测时活动/展位/商品数据创建操作链路长、耗时、失败率高

### 稀缺度定位

#### 核心定位

**具备跨端框架设计能力的大前端架构师**

#### 为什么稀缺

- 大部分前端只做 Web，不碰原生。能同时理解 Web + Native + 桌面端的人本身就是少数
- 做过跨端框架的人更少。用 RN/Flutter 的人多，理解框架底层设计（Bridge、渲染引擎、JSI）的人很少
- 工程化 + 性能 + 质量的全链路能力需要多年积累
- 需要既懂技术深度又能做技术决策，要求在多个终端形态上都有实战经验

#### 终端覆盖（五端实战）

| 终端形态 | 对应项目 |
|---------|---------|
| Web | 全链路性能分析平台、MT 优选、打车 H5 |
| App（RN） | XC 机酒、XT App |
| 小程序 | MT 优选客户端 |
| 桌面端（Electron） | AIOT-IDE（基于 VS Code，含 LSP、DAP、依赖分析引擎） |
| 跨端框架层 | 快应用框架（Android 原生 + 跨端方案设计） |

#### 叙事线（把跳槽变成能力升级路径）

> Web 基础扎实（MT/DFGX）→ 跨端应用开发（RN/小程序，XC/XT）→ 跨端框架设计（快应用框架，XM）→ 端到云全链路平台建设（性能分析平台，XM）

#### 面试核心打法

1. **我不只是用跨端框架，我还造过** — 快应用框架的设计与实现
2. **我不只是做业务，我还建过平台** — 全链路性能分析平台、低代码搭建系统、AIOT-IDE
3. **我不只是写前端，我还懂原生** — Android/NDK/JSI、RN Bridge、条件编译
4. **我不只是做单点优化，我还推过全链路治理** — 工程化 + CI/CD + 质量门禁 + 监控运维
