# XRN页面加载流程
- 页面跳转 => 创建容器 - 增量包检测 - 框架加载 - 业务js加载 - 首屏首次渲染 - 首屏网络请求 - 首屏二次渲染

# 页面加载流程解析
- 增量包检测：对跳转的业务频道进行增量包检测，如果有增量，下载增量包 - 解压 - 安装：得到jsbundle
- 业务jsbundle时，会执行页面生命周期
- 二次渲染： net-res => state => view变化

# 针对页面加载流程的优化
- 增量包：预下载
- 框架可以后台预加载
- 业务js懒加载 预加载
    - 尽量使用lazyRequire加载页面js ： 避免首次加载量太大
    - 业务模块仅使用hermes : 
- 网络请求：预请求 并发请求 缓存
