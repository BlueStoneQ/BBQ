## 一、启动流程（按优先级排序）

### 1.1 第一步：搞定钱的链路

在写一行代码之前，先确保你能收到钱：

1. **开港卡**（最优先）
   - 推荐：招商永隆 / 招商香港一卡通 / 中银香港
   - 线上开户可选：ZA Bank（众安银行），门槛最低
   - 需要：身份证 + 港澳通行证 + 地址证明
   - 用途：收 Steam 分成、支付海外服务（域名、工具订阅、广告投放）

2. **注册 Steamworks 开发者账号**
   - 费用：100 美元（一次性，港卡直接支付）
   - 地址：https://partner.steamgames.com
   - 填写银行电汇信息（港卡的 SWIFT Code + 账户号）

3. **税务设置**
   - 填写 W-8BEN 表格（美国税务，中国居民适用）
   - 中美税收协定可将预扣税从 30% 降到 10%
   - 国内收入需自行申报个人所得税
   - 每人每年 5 万美元等值结汇额度，初期够用

4. **资金回流**
   - Steam 每月打款一次（达到 100 美元门槛）
   - 港卡收美元 → 结汇 → 转账回大陆银行卡
   - 收入规模大了建议注册香港公司或大陆公司，走对公账户

### 1.2 第二步：确定游戏方向

- 选择一个 3-6 个月能完成的小体量项目
- 适合独立开发者的类型：Roguelike、平台跳跃、塔防、视觉小说、卡牌、解谜
- 写一份简单的 GDD（游戏设计文档），明确核心玩法

### 1.3 第三步：搭建开发环境

- 引擎选择（见下文详细对比）
- Git + GitHub 版本控制
- Kiro / Cursor 作为 AI 辅助 IDE
- 注册必要的 AI 工具账号

## 二、为什么选买断制

| 维度 | 买断制 | F2P（免费+内购） |
|------|--------|-----------------|
| 服务器需求 | ❌ 不需要 | ✅ 必须 |
| 运维成本 | 极低 | 高 |
| 收入模型 | 简单透明 | 复杂，需要持续运营 |
| 开发周期 | 可控 | 需要持续更新 |
| 适合团队规模 | 1-3 人 | 通常需要 5+ 人 |
| 合规要求 | 出海不需要版号 | 国内需要版号 |

**结论**：没有服务器资源 + 独立开发者 + 出海 = 买断制是最优解。

## 三、引擎选择

| 引擎 | 语言 | 适合场景 | AI 代码生成质量 | 费用 |
|------|------|---------|---------------|------|
| Unity | C# | 2D/3D 通用 | ⭐⭐⭐⭐⭐ | 免费（注意 Runtime Fee 政策） |
| Godot | GDScript/C# | 2D 优秀，3D 在进步 | ⭐⭐⭐⭐ | 完全免费开源 |
| Unreal | C++/Blueprint | 3A 级 3D | ⭐⭐⭐ | 免费（流水超 100 万美元收 5%） |
| Cocos Creator | TypeScript | 2D/轻 3D | ⭐⭐⭐ | 免费 |

**推荐**：
- 前端背景 → Godot（GDScript 学习曲线平缓）或 Cocos Creator（TypeScript 直接上手）
- 追求生态和资源 → Unity
- 追求画面 → Unreal

## 四、AI 全流程开发链路

### 4.1 策划阶段

| 任务 | AI 工具 | 用法 |
|------|---------|------|
| 头脑风暴 | Claude / ChatGPT | 输入游戏类型和灵感，让 AI 发散创意 |
| GDD 撰写 | Claude | 给出核心玩法描述，AI 生成完整 GDD 框架 |
| 数值设计 | ChatGPT + 表格 | AI 推演平衡性，生成数值表 |
| 流程图 | Kiro + Mermaid | 在 IDE 里直接生成游戏流程图、状态机 |
| 竞品分析 | AI + Web Search | 分析同类游戏的 Steam 评价和玩法特点 |

### 4.2 编码阶段

**核心工作流**：

```
需求定义（Spec）→ AI 生成代码 → 人工 Review → 调试优化 → 提交
```

| 任务 | 工具 | 说明 |
|------|------|------|
| 代码编写 | Kiro | Spec 驱动开发，按系统拆分任务 |
| 代码补全 | GitHub Copilot / Cursor | 实时补全，提高编码速度 |
| 架构设计 | Claude | 讨论 ECS、状态机等游戏架构方案 |
| Shader 编写 | AI + 引擎文档 | 描述效果，AI 生成 Shader 代码 |
| Bug 修复 | Kiro | 贴报错信息，AI 定位和修复 |

**Kiro Spec 驱动开发示例**：

把游戏拆成独立系统，每个系统一个 Spec：
- `spec/combat-system.md` — 战斗系统
- `spec/inventory-system.md` — 背包系统
- `spec/dialogue-system.md` — 对话系统
- `spec/save-system.md` — 存档系统
- `spec/audio-system.md` — 音频系统

每个 Spec 包含：需求描述 → 技术设计 → 任务清单 → 逐个让 AI 实现

### 4.3 美术阶段

#### 2D 美术

| 任务 | 工具 | 说明 |
|------|------|------|
| 概念设计 | Midjourney / Stable Diffusion | 文字描述生成概念图 |
| 角色立绘 | SD + ControlNet | 保持角色一致性 |
| 像素美术 | Aseprite + AI 草图 | AI 出草图，手动像素化精修 |
| 场景/背景 | SD + img2img | 草图转精细场景 |
| UI 设计 | Figma + AI 插件 | 快速出 UI 方案 |
| 抠图 | Remove.bg / Rembg | 自动去背景 |
| 风格一致性 | SD LoRA / IP-Adapter | 训练或使用风格模型锁定画风 |

#### 3D 美术

| 任务 | 工具 | 说明 |
|------|------|------|
| 模型生成 | Meshy / Tripo3D | 文字/图片 → 3D 模型 |
| 模型精修 | Blender + AI 插件 | 拓扑优化、UV 展开 |
| 材质贴图 | Stable Diffusion | 生成 PBR 贴图 |
| 天空盒 | Skybox AI | 全景天空盒生成 |
| 动画 | Mixamo / AI Motion | 自动绑骨和动画 |

#### 美术核心原则

1. **先定风格再生产** — 确定参考图，用 LoRA 锁定风格后再批量生成
2. **AI 生成 → 筛选 → 精修** — 不要指望一次出成品
3. **注意商用授权** — Midjourney 付费版可商用，SD 开源模型检查 License

### 4.4 音频阶段

| 任务 | 工具 | 说明 |
|------|------|------|
| BGM | Suno / Udio / Soundraw | 描述风格生成音乐 |
| 音效 | Stability Audio / Freesound | AI 生成 + 免费素材库 |
| 配音 | ElevenLabs | NPC 对话语音合成 |
| 音频集成 | FMOD / Wwise | 游戏音频中间件（免费额度够用） |

**注意**：AI 生成音乐的版权政策还在演变中，商用前确认平台最新条款。

### 4.5 文案与本地化

| 任务 | 工具 | 说明 |
|------|------|------|
| 剧情/对话 | Claude | 质量最好，擅长长文本和角色塑造 |
| 物品/技能描述 | ChatGPT / Claude | 批量生成，保持风格统一 |
| 翻译 | Claude / GPT-4 | 初翻质量已经很好 |
| 文本管理 | JSON / CSV | 所有文本外置，方便批量翻译替换 |
| Steam 商店页面 | AI 多语言生成 | 至少英文 + 简中 |

### 4.6 测试阶段

| 任务 | 工具 | 说明 |
|------|------|------|
| 单元测试 | Kiro 自动生成 | AI 辅助写测试用例 |
| Bug 分析 | AI + 日志 | 贴 crash log，AI 定位问题 |
| 玩家测试 | Steam Playtest | 免费的封闭测试功能 |
| 反馈分析 | AI 分类 | 批量处理玩家反馈，排优先级 |

## 六、发行与运营

### 6.1 Steam 上架流程

1. 创建应用（App）→ 填写商店信息
2. 上传商店素材（胶囊图、截图、预告片）
3. 设置价格（建议参考同类游戏定价，首发可以打折）
4. 提交审核（通常 2-5 个工作日）
5. 设置发售日期

### 6.2 关键时间节点

| 节点 | 行动 |
|------|------|
| 开发初期 | 上 Steam 商店页面，开始积累愿望单 |
| 发售前 3-6 个月 | 参加 Steam Next Fest，发布免费 Demo |
| 发售前 1 个月 | 社交媒体预热（Twitter/X、Reddit、Discord） |
| 发售日 | 首发折扣（10-15%），最大化首周销量 |
| 发售后 | 持续更新，回应社区反馈 |

### 6.3 市场推广（AI 辅助）

| 任务 | 工具 |
|------|------|
| 宣传文案 | Claude / ChatGPT 多语言生成 |
| 宣传图 | Midjourney / SD 生成素材 |
| 预告片脚本 | AI 撰写 + 剪辑指导 |
| 社区运营 | AI 辅助回复 Discord / Steam 论坛 |
| devlog | AI 辅助撰写开发日志 |

## 七、核心问题解决方案

### Q1：没有美术基础怎么办？

- 选择像素风 / Low-Poly / 极简风格，降低美术门槛
- AI 生成 + 手动微调，不需要从零画
- Asset Store / itch.io 购买基础素材包，AI 做风格统一化处理
- 考虑和一个美术合作，用收入分成模式

### Q2：一个人开发周期太长？

- 控制项目规模，3-6 个月能完成的体量
- 用 AI 把重复性工作自动化（代码生成、素材批量处理、文本翻译）
- 先做 Vertical Slice（垂直切片），验证核心玩法再铺量
- 不要追求完美，先发售再迭代

### Q3：如何保持 AI 生成内容的一致性？

- 美术：训练 LoRA 模型或使用 IP-Adapter 锁定风格
- 文案：给 AI 一份风格指南（语气、用词、世界观设定）
- 代码：用 Kiro Steering 文件定义编码规范，AI 自动遵循
- 音乐：固定 Suno 的风格标签和参考曲目

### Q4：版权和法律风险？

- AI 生成内容的版权归属目前各国法律不同，但实际操作中：
  - Midjourney 付费版：可商用
  - Stable Diffusion 开源模型：大部分可商用（检查具体 License）
  - Suno 付费版：可商用
  - ElevenLabs 付费版：可商用
- 不要用 AI 生成明显模仿已有 IP 的内容
- 建议在游戏 Credits 中注明 AI 辅助工具

### Q5：Steam 审核会因为 AI 内容被拒吗？

- Steam 2023 年底已更新政策，允许 AI 生成内容上架
- 需要在提交时披露 AI 使用情况
- 确保 AI 生成内容不侵犯他人版权即可

### Q6：如何定价？

- 参考同类型、同体量游戏的定价
- 独立小游戏通常 $4.99 - $14.99
- 中等体量 $14.99 - $29.99
- 首发折扣 10-15% 能有效提升首周销量
- 中国区定价 Steam 会自动建议区域定价
